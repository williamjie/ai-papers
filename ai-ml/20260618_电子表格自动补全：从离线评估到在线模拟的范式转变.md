# ⭐ 电子表格自动补全：从离线评估到在线模拟的范式转变

**日期**: 2026-06-18

---

论文 : A Benchmark and Framework for Evaluating Next Action Predictions in Spreadsheets链接 : https://arxiv.org/abs/2606.13802代码自动补全（Code Completion）已经彻底改变了开发者的效率，但在电子表格领域，尽管用户每天执行成千上万次格式化操作，通用的动作预测助手却几乎空白。这篇论文不仅填补了这一空白，更关键的是它提出了一套 在线评估框架（Online Evaluation Framework） ，解决了传统离线评估无法反映真实交互后果的痛点。
### 为什么现有方案行不通？
在代码领域，我们有详细的版本历史（Git History），但在电子表格中，公开语料库通常只保留最终状态，缺乏中间编辑轨迹。
更深层的问题在于评估方式。传统的“给定当前状态预测下一步”是离线评估（Offline Evaluation）。这种评估假设每次预测都是独立的，且不会改变后续状态。然而在真实场景中：
- 状态累积误差：如果模型预测错误并被用户接受，后续所有预测都基于这个错误状态，导致误差雪崩。
- 重复得分陷阱：离线评估可能让模型反复预测简单的动作来刷高准确率，而在在线交互中，用户不会接受无意义的重复建议。
### 核心方法拆解：构建真实世界的模拟器论文的核心贡献在于构建了一个名为 NAPE 的基准测试和在线评估框架。其设计直觉非常工程化： 评估必须模拟用户的“接受/拒绝”行为及其对剩余任务的影响。
#### 1. 数据构造：从静态到动态由于缺乏现成的编辑历史，作者通过“启发式种子 + LLM 细化 + 人工标注”三阶段流水线，从公开语料库中逆向重构了 52 个电子表格的创建轨迹。
- 规模：共 11,907 次操作，平均每个表格 229 步。
- 多样性：覆盖了输入、合并、字体、边框、对齐等 9 类原子操作。
#### 2. 在线评估循环（Online Evaluation Loop）
这是论文最精彩的部分。评估不再是简单的 P(y∣x)P(y|x) ，而是一个动态过程：
- 预测：模型根据历史 HH 预测一组动作 PP。
- 决策：基于启发式规则（如精度、节省步数）决定是否接受 PP。
- 状态更新：
若接受：从未来任务队列中移除已完成的动作，并插入逆操作以撤销错误预测带来的副作用（例如填错了颜色需要额外一步清空）。
- 若拒绝：用户执行下一步，历史 HH 增加一条记录。
这种设计迫使模型不仅要“猜得准”，还要“懂得适可而止”。如果模型为了刷准确率而输出大量低置信度动作，会导致用户需要花费更多时间撤销，最终导致**用户节省操作数（User Actions Saved, UAS）**为负。
### 关键实验结果论文对比了零样本 LLM、微调小语言模型（SLM）和经典机器学习模型。以下是核心发现：
模型 UAS (单动作重预测) 精度 (PREC) 接受率 (AR) GPT-5-R (推理模式) 32.7% 41.6% 29.4% FT-SmolLM2-360M (微调) 26.8% 33.7% 26.8% GPT-5 (非推理) 27.4% 44.8% 30.9% GPT-5 mini 18.0% 21.9% 16.8% Online n-gram (无训练) 12.0% 20.4% 14.7%反直觉发现 1：拒绝比接受更重要当使用“总是接受（ALWAYS）”策略时，UAS 降至 -19.2% 。这意味着如果模型不懂何时闭嘴，它的建议不仅无用，反而会成为用户的负担。这证明了**弃权机制（Abstention）**在自动补全系统中的核心地位。
反直觉发现 2：小模型微调可媲美大模型经过监督微调（SFT）的 360M 参数 SmolLM2 模型，其 UAS (26.8%) 已接近 GPT-5 (27.4%)。这表明电子表格动作预测是一个高度结构化的任务，领域特定的小模型足以胜任，无需依赖昂贵的超大参数模型。
### 工程启示-触发策略（Trigger）至关重要实验显示，每步都触发预测（Stride=1）的 UAS (27.4%) 远高于每 8 步触发一次 (10.6%)。这意味着我们需要轻量级的本地模型或高效的触发器，以便在用户每次操作后都能实时评估是否介入。
-上下文窗口的边际效益递减将上下文从 32 步增加到 128 步，UAS 从 27.4% 提升至 30.0%；但继续增加到 512 步，提升微乎其微。工程上无需盲目追求超长上下文，保留最近 128 个操作足以捕捉大部分模式。
-预测长度的自我调节限制模型每次最多预测 16 个动作反而降低了 UAS。这说明优秀的模型能够根据置信度自主决定预测序列的长度（Stop Token 机制）。强制截断会破坏高置信度的长链预测，而过度放宽则引入噪音。
### 局限与展望目前基准仅包含 52 个表格，虽然质量高但规模有限。此外，评估依赖于启发式接受规则，而非真实人类用户的行为数据。未来工作应关注如何收集真实的用户交互日志，以及探索多模态输入（如截图）对预测精度的提升作用。
这篇论文为电子表格 Agent 的开发提供了一套严谨的评估标准。它提醒我们：在构建自动化助手时， “不做错事”往往比“做对更多事”更能提升用户体验。
