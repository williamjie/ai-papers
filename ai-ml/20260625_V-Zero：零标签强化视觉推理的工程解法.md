# ⭐⭐½ V-Zero：零标签强化视觉推理的工程解法

**日期**: 2026-06-25

论文 : V-Zero: Answer-Label-Free On-Policy Distillation with Contrastive Evidence Gating for Fine-Grained Visual Reasoning链接 : https://arxiv.org/abs/2606.25319在提升多模态大模型（MLLMs）的细粒度视觉推理能力时，我们常陷入两难：强化学习（RL）训练成本极高且难以调优，而监督微调（SFT）又极度依赖昂贵的标注数据。
这篇来自四川大学和北京大学的论文提出了 V-Zero ，一个无需任何文本答案标签、仅靠对比视觉证据就能实现高效蒸馏的框架。它不仅避开了 RL 的高昂探索成本，还在推理时保持了标准的全图输入模式，工程落地价值极高。
### 痛点：现有方案的“不可能三角”
目前的细粒度视觉推理主要依赖两种路径，但都有明显缺陷：
- 基于代理（Agentic）的视觉搜索：如 DeepEyes、Thyme 等，通过 RL 让模型学会“看哪里”。这需要定义可验证的奖励函数，探索成本高，且推理时需要额外的工具调用开销。
- 大规模 SFT：如 ZwZ，通过在标注好的图像-文本对上进行微调。这需要海量高质量数据，且容易导致模型灾难性遗忘原有能力。
核心问题在于： 能否在不使用昂贵的 RL 探索、不依赖大规模文本标签、且不破坏模型原有能力的情况下，提升视觉推理？
作者给出的答案是肯定的，关键在于重新审视 On-Policy Distillation (OPD) 。
### 方法拆解：从“无负样本对齐”到“证据门控”
V-Zero 的核心创新在于对 OPD 的理论重构以及引入的**对比证据门控（Contrastive Evidence Gating）**机制。
1. 重新理解 OPD 的局限性传统 OPD 让学生模型自己采样轨迹，然后让教师模型在相同前缀下提供 token 级修正。作者将其重新解释为“无负样本的停止梯度对齐”。
- Insight：OPD 能提供密集的 token 级监督，但缺乏轨迹级别的判别能力。一旦学生走入错误路径，教师只能基于该错误前缀提供局部修正，无法判断整个轨迹是否偏离了正确答案。
2. 对比证据门控机制为了解决上述问题，V-Zero 引入了教师端的“视觉证据对比”模块：
- 正样本视图（Positive View）：包含任务相关区域的裁剪图（Crop）。
- 负样本视图（Negative View）：从原图 2x 下采样后随机采样的无关区域。
在训练时，学生模型基于全图采样一组“兄弟轨迹”（Sibling Rollouts）。教师模型会分别在这些正负视图下回放这些轨迹，计算每个 token 的对数概率差值 Δk\Delta_k ​ 。
- 如果某个 token 在正视图下的概率远高于负视图，说明该推理步骤强依赖于正确的视觉证据。
- 将这些差值聚合为轨迹级别的证据优势（Evidence Advantage），并归一化为门控权重 w(g)w^{(g)}。
最终，只有那些被“视觉证据”强烈支持的轨迹，其对应的 OPD 损失才会被加权放大；反之则被抑制。这就实现了无需文本标签的、基于视觉依赖性的轨迹筛选。

### 工程启示- 摆脱对文本标签的依赖：V-Zero 证明，通过构造简单的正负视觉视图（Crop vs Random Crop），就可以生成高质量的训练信号。这大幅降低了数据准备门槛，无需人工标注推理轨迹或最终答案。
- 推理零开销：与 Agentic 方法不同，V-Zero 仅在训练阶段使用裁剪图进行门控计算。在推理时，模型依然接收全图输入，保持了标准的 MLLM 接口，易于集成到现有系统中。
- OPD 的实用化路径：对于希望利用 OPD 但受限于其“缺乏轨迹判别”缺陷的团队，V-Zero 提供了一种通用的思路：引入额外的对比信号来评估采样轨迹的质量，从而指导蒸馏过程。
⚠️ 局限与注意- V-Zero 的性能提升依赖于教师模型（27B）的质量和学生模型（4B）的规模差距。消融实验显示，使用较小的 9B 教师模型效果略逊于 27B 教师。
- 在 HR-8K 等高分辨率基准上，由于全图信息已经足够丰富，对比证据门控的收益相对较小（+2.0），说明该方法在信息极度受限或需要精细定位的场景下优势更明显。
V-Zero 为细粒度视觉推理提供了一个高效、低成本的解决方案，特别适合那些希望在不增加推理延迟和数据标注成本的前提下，快速提升模型视觉细节理解能力的工程团队。
