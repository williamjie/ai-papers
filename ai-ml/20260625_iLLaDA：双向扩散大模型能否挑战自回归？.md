# ⭐⭐½ iLLaDA：双向扩散大模型能否挑战自回归？

**日期**: 2026-06-25

论文 : Improved Large Language Diffusion Models链接 : https://arxiv.org/abs/2606.25331当前的大语言模型（LLM）几乎被自回归（Autoregressive, AR）范式垄断，但扩散模型（Diffusion Models）在文本生成上的潜力一直未被充分挖掘。这篇论文推出了 iLLaDA，一个从 scratch 训练的 8B 参数双向掩码扩散语言模型。它不仅在预训练规模上达到了 12T tokens，更通过一系列工程优化，在多个基准测试中逼近甚至超越了同量级的 Qwen2.5 7B。这证明了非自回归架构并非只是学术玩具，而是具备实际竞争力的技术路径。
### 为什么需要重新审视扩散模型？
自回归模型虽然强大，但存在固有的顺序依赖问题，导致推理速度受限且难以并行。之前的 LLaDA 展示了双向扩散模型可以具备上下文学习和指令跟随能力，但其性能仍落后于强力的自回归基线如 Qwen2.5。
iLLaDA 的核心动机是填补这一差距。作者认为，通过大规模预训练和针对性的后训练策略优化，完全可以从零开始训练出高性能的双向扩散模型，而无需依赖自回归模型的权重初始化。这不仅是架构上的探索，更是对“语言智能是否必须依赖自回归”这一常识的挑战。
### 核心方法拆解：不仅仅是换了个 LossiLLaDA 的设计细节充满了工程直觉，以下是几个关键改进点：
-架构优化与参数效率GQA 引入：不同于原版 LLaDA 使用多头注意力，iLLaDA 采用分组查询注意力（Grouped-Query Attention, GQA）。这是因为扩散模型在推理时可以利用类似 KV-Cache 的机制，GQA 能显著降低缓存 Key/Value 状态的内存占用。
- Embedding 共享：输入和输出 Embedding 矩阵被绑定（Tied），进一步减少了参数量。尽管总参数量为 8B，但非嵌入参数仅为 6.98B，与 LLaDA 一致，但架构更轻量。
-训练策略的彻底重构预训练数据量翻倍：从 LLaDA 的 2.3T tokens 激增至 12T tokens。学习率策略也进行了调整，先线性预热至 2×10−42 \times 10^{-4}10−4 并保持恒定，当损失不再下降时切换为余弦衰减，最小学习率为 5×10−65 \times 10^{-6}10−6。
- SFT 的“全序列掩码”：这是最关键的设计变化。传统 SFT 通常只 Mask 回答部分，而 iLLaDA 在监督微调（Supervised Fine-Tuning, SFT）阶段沿用了预训练的数据处理方式。它将指令、回答和 EOS 拼接成连续序列，并对整个序列应用随机掩码。这种一致性使得模型在微调时能更好地利用双向上下文信息。
- 长周期微调：在 25B tokens 的指令集上进行了 12 个 Epoch 的微调。消融实验显示，扩散模型在数据受限场景下具有极强的“数据复用”能力，性能随 Epoch 增加持续提升，这与自回归模型容易过拟合的特性不同。
-推理与评估创新变长块生成：采用可变长度的块生成策略，每次采样一个 Mask 块，将高置信度预测转为可见 token，低置信度继续 Mask，直到出现 EOS 或达到预算。这提高了生成效率。
- 基于置信度的评分：对于多选题评估，传统的似然估计并不准确。iLLaDA 引入了一种确定性置信度评分规则：从全 Mask 状态开始，每一步选择模型最自信的 token 进行揭示，累加对数概率作为得分。实验表明，这种方法在 PIQA、ARC-C 等任务上显著优于传统方法。
### 关键结果：数据不会撒谎iLLaDA 的表现令人印象深刻，尤其是在基础模型（Base）层面，它已经具备了与顶级自回归模型掰手腕的实力。

反直觉发现 ：扩散模型在 SFT 阶段表现出极强的“数据饥渴”和复用能力。论文指出，即使在较小的指令集上进行多达 12 个 Epoch 的训练，性能依然持续上升，没有出现明显的过拟合迹象。这暗示了扩散目标函数在利用有限高质量数据方面的独特优势。
### 工程启示与局限对于工程师而言，iLLaDA 提供了几个重要启示：
- 双向上下文的价值：在处理长文档理解、复杂推理或多模态任务时，双向注意力机制可能比自回归更具优势，因为它能同时利用前后文信息。
- SFT 策略需匹配架构：不要直接套用自回归模型的 SFT 流程。对于扩散模型，保持预训练和微调数据分布的一致性（如全序列掩码）至关重要。
- 评估方法的适配：传统的似然评分不适合扩散模型，基于置信度的逐步揭示评分是更公平且有效的评估手段。
当然，iLLaDA 仍有局限。目前尚未引入强化学习（RLHF/RLAIF），这是其与顶级自回归模型的主要差距来源。此外，变长块生成虽然高效，但在某些代码生成任务（如 HumanEval）上，过长的块长度可能会损害性能，需要精细调参。
总的来说，iLLaDA 证明了双向扩散大模型是一条可行且强大的技术路线。随着推理加速技术和对齐方法的成熟，我们或许很快会看到更多非自回归架构在实际生产环境中的落地。
