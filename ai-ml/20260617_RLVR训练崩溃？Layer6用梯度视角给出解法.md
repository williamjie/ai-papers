# ⭐⭐ RLVR训练崩溃？Layer6用梯度视角给出解法

**日期**: 2026-06-17

论文 : A Gradient Perspective on RLVR Stability and Winner Advantage Policy Optimization链接 : https://arxiv.org/abs/2606.16154RLVR（Reinforcement Learning with Verifiable Rewards）正在成为提升大模型推理能力的标配，但 GRPO 类算法在训练后期经常发生“崩溃”：模型要么输出无意义的乱码（高熵崩溃），要么陷入重复的僵化模式（低熵崩溃）。
Layer6 AI 这篇新论文没有停留在调参层面，而是从 Token 级梯度动力学 的角度，彻底拆解了崩溃的微观机制，并提出了一种极简且有效的改进方案：WAPO。
### 为什么现有的“信任域”救不了你？
主流观点认为，RLVR 的不稳定源于旧策略 πθold\pi_{\theta_{old}} ​ ​ 与新策略 πθ\pi_\theta ​ 之间的分布漂移（Drift）。因此，DAPO、GSPO 等方法试图通过更复杂的裁剪（Clipping）或重要性采样来限制这种漂移。
但作者发现了一个反直觉的现象： 单纯限制偏离度并不能保证稳定性。
⚠️ 关键发现 ：在 OTT-QA 数据集上训练 SmolLM3-3B 时，即使使用 DAPO 并加大裁剪力度（ ϵlow=0.9\epsilon_{low}=0.9 ​ = 0.9 ）以抑制发散 Token，训练依然会崩溃。更糟糕的是，这种激进裁剪反而会导致熵增加，加速模型退化。
这说明，“离旧策略有多远”并不是预测稳定性的可靠指标。我们需要换个视角： 梯度更新究竟如何改变了局部概率分布？
### 核心 Insight：Peak-Valley 梯度分类法作者推导了一阶梯度近似公式，发现 Token 的更新方向不仅取决于优势值（Advantage）的正负，还取决于该 Token 在当前策略下的概率高低。由此定义了两个关键概念：
- Peak Tokens：采样概率高于自适应阈值 C(p)C(p) 的高频词。
- Valley Tokens：采样概率低于阈值的低频词。
结合优势值符号（Pos/Neg），形成四个象限：
- Pos-Peak：强化高频正确路径。效果：稳定，但保守，容易过早收敛。
- Neg-Valley：抑制低频错误路径。效果：初期奖励提升快，但后期易导致低熵崩溃（模型变得过度自信且重复）。
- Pos-Valley & Neg-Peak：这是不稳定的根源。这两类更新都会增加熵。Pos-Valley 试图强化罕见正确路径，却意外抬高了其他无关 Token 的概率；Neg-Peak 抑制高频路径，导致概率质量重新分布到随机区域。
### WAPO：只让“赢家”说话基于上述分析，作者提出了 Winner Advantage Policy Optimization (WAPO) 。
其核心逻辑极其简单： 在策略梯度更新中，直接掩码（Mask）掉所有非正优势值（Non-positive Advantage）的完成序列。
这听起来像极了 Rejection Fine-Tuning，但 WAPO 保留了在线策略梯度的核心特性：
- 保留重要性比率：仍然使用 πθ/πθold\pi_\theta / \pi_{\theta_{old}}​/πθold​​。
- 组内归一化：优势值仍在 Group 内部计算，保持相对性。
- 自适应学习率因子：理论证明，WAPO 的更新方向与二元奖励下的策略梯度一致，但带有一个自适应因子 (1−qx)(1 - q_x)qx​)。这意味着对于已经掌握的问题（qx→1q_x \to 1​→1），更新强度自动减弱；对于难题（qx→0q_x \to 0​→0），保持探索压力。
### 实验结果：简单即有效作者在数学推理（NuminaMath-LEAN, Math-500）和多跳问答（Hotpot-QA, OTT-QA）上，对比了 GRPO、DAPO、GSPO 和 WAPO。
1. 稳定性碾压在 NuminaMath-LEAN 上，GRPO 使用 Gemma3-4B 时直接崩溃，而 WAPO 全程稳定。在多跳 QA 任务中，WAPO 的优势更为明显：
- OTT-QA (Qwen3-4B)：WAPO 比最强基线高出 9.9%。
- Hotpot-QA (Gemma3-4B)：WAPO 领先基线 10.6%。
2. 泛化能力更强在 Out-of-Domain 测试中（Hotpot-QA -> 2-wiki），WAPO 在所有三个模型家族（Qwen, SmolLM, Gemma）上均取得最佳 Exact Match 分数。例如 Qwen3-4B 达到 34.2% ，显著优于 DAPO (13.3%) 和 GRPO (26.6%)。
3. 探索性并未丧失有人担心只更新正样本会导致过拟合。但 Pass@k 数据显示，WAPO 在保持高 Pass@1 的同时，Pass@32 的表现同样强劲（SmolLM3-3B 上达到 66.7%，与 GSPO 持平），证明其并未牺牲多样性。
### 工程启示对于正在落地 RLVR 的工程师，这篇论文提供了两个极具价值的建议：
- 不要过度迷信复杂的裁剪机制：如果训练不稳定，尝试简化目标函数。WAPO 的代码修改量极小，只需在计算 Loss 前过滤掉 Ai≤0A_i \le 0​≤0 的样本即可。
- 关注 Token 级的熵变化：监控训练过程中的 Per-token Entropy 比监控 KL Divergence 更能早期发现崩溃迹象。如果熵异常升高且奖励下降，通常是 Neg-Peak 或 Pos-Valley 更新失控所致。
WAPO 证明了在 RLVR 中，“少即是多”。通过剔除噪声巨大的负样本梯度，我们不仅获得了稳定性，还意外提升了模型在复杂推理任务上的上限。
