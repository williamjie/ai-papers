# 视频物理评测新范式：PQSG图结构拆解

**日期**: 2026-06-26

---

论文 : Physics Question Scene Graph: Fine-grained Evaluation of Physical Plausibility in Text-to-Video Generation链接 : https://arxiv.org/abs/2606.25306现在的文生视频模型（如 Sora, Veo）在视觉逼真度上已经卷无可卷，但“物理规律”依然是阿喀琉斯之踵。
更头疼的是，现有的评测指标往往只给一个笼统的分数，无法告诉工程师到底是物体没生成对，还是动作逻辑崩坏，亦或是违反了重力法则。
这篇论文提出的 PQSG（Physics Question Scene Graph） 提供了一种基于有向无环图（DAG）的细粒度评测框架，不仅给出了分数，更精准定位了“死因”。
### 为什么现有方案不够用？
传统评测方法（如 VideoScore, PhyGenEval）通常将视频质量压缩为一个标量。这种“黑盒”评分有两个致命缺陷：
- 缺乏可解释性：你只知道视频分低，但不知道是物体缺失还是物理交互错误。
- 逻辑依赖被忽略：评测往往独立判断各个属性。例如，如果视频中根本没有球，评测系统却去问“球的落体加速度是否合理”，这会导致幻觉或误导性评分。
核心 Insight ：物理评测必须遵循逻辑依赖链。只有当“物体存在”且“执行了正确动作”后，才能评估其“物理交互的合理性”。
### 方法拆解：PQSG 的图结构设计PQSG 的核心创新在于将评测问题组织成一个层级化的图结构，包含三个关键节点类型：
- Object（物体）：验证提示词中的关键对象是否存在（如：“桌上有两个枕头吗？”）。
- Action（动作）：验证对象是否执行了指定动作（如：“夹子释放了网球吗？”）。
- Physics（物理）：评估动作是否符合物理常识（如：“枕头在撞击下可见变形吗？”）。
关键设计直觉：
PQSG 通过边（Edges）建立了严格的父子依赖关系。如果父节点回答为“否”，所有子节点自动标记为“无效/否”。
- 示例：如果物体节点判定“没有橙色方块”，那么后续关于“方块下落”和“方块反弹”的物理问题直接跳过，不再查询 VLM。
- 价值：这不仅减少了无效推理带来的幻觉，还确保了评分的逻辑严密性。只有当前提条件满足时，物理评测才生效。
### 关键结果：细粒度胜过整体打分作者在 FinePhyEval 数据集上进行了验证，该数据集包含 195 个人工标注的视频-提示对，涵盖 Sora 2, Veo 3, Wan 2.1 等主流模型生成的视频。
1. 与人类判断的相关性更高PQSG 在整体评分上与人类 Likert 评分的相关性显著优于基线方法：
指标 Pearson’s r Kendall’s τ Spearman’s ρ VideoScore 0.289 0.262 0.378 PhyGenEval 0.272 0.220 0.264 PQSG (GPT-5.5) 0.478 0.336 0.456反直觉发现 ：人类在打分时，对“物理合理性”的敏感度远高于物体或动作。数据显示，物理类别与整体评分的相关系数高达 0.85，而物体仅为 0.44。这意味着，如果视频物理逻辑崩坏，人类会直接给低分，无论画面多清晰。
2. 模型排名：闭源依然碾压开源利用 PQSG 对主流模型进行细粒度打分，结果如下：
模型 Object Score Action Score Physics Score Overall Score Sora 2 0.95 0.75 0.69 0.78 Veo 3 0.98 0.78 0.68 0.80 Wan 2.1 0.86 0.53 0.46 0.59 Cosmos 14B 0.93 0.56 0.46 0.62可以看出，所有模型在 Object 维度都表现极佳（>0.86），但在 Action 和 Physics 维度大幅下滑。尤其是开源模型 Wan 2.1 和专门针对物理模拟设计的 Cosmos，在物理交互上得分均低于 0.5。
3. VLM 的 QA 瓶颈虽然 VLM（如 GPT-5.5）在生成问题（QG）时准确率极高（Precision >95%），但在回答问题（QA）时存在明显短板：
- Object QA 准确率：~88%- Physics QA 准确率：~65%VLM 在处理快速动作和复杂物理交互（如烟雾扩散、液体吸收）时，容易表现出“Yes-bias”（倾向于回答是），导致对物理违规的漏判。
### 工程启示- 评测即调试工具：不要只追求一个总分。使用 PQSG 这样的图结构评测，可以快速定位模型是在“生成物体”还是“模拟物理”上失效，从而指导你是需要调整 Text-Encoder 还是优化扩散模型的时空注意力机制。
- 依赖链的重要性：在设计任何自动化评测 Agent 时，务必引入逻辑前置检查。避免在前提不成立的情况下进行深层推理，这能显著降低幻觉率。
- 物理评测的瓶颈在 VLM：目前的瓶颈不在于如何提问，而在于 VLM 本身缺乏足够的时空物理推理能力。未来改进方向可能是结合专门的物理模拟器或强化学习奖励模型来辅助 QA 阶段。
### 局限与展望PQSG 目前依赖强大的闭源 VLM（GPT-5.5, Gemini）进行问答，成本较高且存在延迟。此外，对于极短的视频片段，VLM 的帧采样率可能不足以捕捉瞬态物理现象（如碰撞瞬间）。
随着多模态模型对时空推理能力的提升，PQSG 的上限还有很大空间。对于工程师而言，这是一个将“模糊的物理直觉”转化为“结构化数据”的优秀范式。
