论文 : SciOrch: Learning to Orchestrate Expert LLMs for Solving Frontier Multimodal Scientific Reasoning Tasks链接 : https://arxiv.org/abs/2606.15872面对前沿科学推理任务，单一模型往往力不从心。SciOrch 提出了一种新范式：训练一个轻量级 8B 模型作为“指挥官”，动态调度多个顶级商业 LLM API 来协同解题。这种方法不仅超越了最强的单体模型，还大幅降低了推理成本，为工程落地提供了极具参考价值的路径。
### 为什么需要“编排”而非“单打独斗”？
当前最强商业模型在 SGI-Reasoning 等前沿科学任务上表现有限，例如 Gemini-3-Pro 准确率仅约 42%。但研究发现，不同模型在不同学科和子任务上存在显著互补性：Claude-Sonnet-4 擅长天文学，o4-mini 强于物理，而 Gemini-3-Pro 在生命科学领域领先。
更关键的是，单一问题往往包含图像解析、领域知识回忆和定量推理等多个子任务，没有任何一个模型能在所有子任务上都做到最好。这意味着，性能提升的空间不在于构建更强的单体模型，而在于如何以细粒度组合现有模型的能力。
### 核心方法：MCTS 驱动的 RL 训练传统的多智能体方法（如自一致性或辩论）协议固定且成本线性增长；路由方法则无法根据中间证据动态调整策略。SciOrch 将科学推理建模为多步决策问题，其核心创新在于解决了“昂贵 API 调用下的强化学习”难题。
- 动作空间设计：编排器在每个步骤选择两种动作之一——委托子问题给特定模型，或提交最终答案。这迫使模型学习何时停止、何时求助，实现选择性路由和早期停止。
- MCTS 生成多样轨迹：由于每次 API 调用都昂贵且耗时，标准在线 RL 不可行。SciOrch 引入蒙特卡洛树搜索（MCTS），通过三阶段策略（多样化展开、信号驱动扩展、剪枝）生成多样化的编排轨迹。
- 节点级训练信号：传统轨迹级梯度噪声大，SciOrch 提取每个节点的单轮样本，利用兄弟节点间的回报差异计算相对优势，结合 REINFORCE++ 进行离线更新。这使得单次 API 调用能产生更多有效训练样本，极大提升了训练效率。
### 实验结果：性能与成本的双赢在包含 240 道题的测试集（涵盖 SGI-Reasoning 和 Scientists’ First Exam）上，SciOrch 展现了显著优势：

### 工程启示与局限对工程实践的启发 ：
- 小模型指挥大模型：无需微调昂贵的顶级模型，只需训练一个轻量级编排器即可利用现有 API 生态，降低了部署门槛。
- 动态路由优于静态集成：通过分解问题和动态选择专家，系统能更精准地匹配任务需求，避免无效计算。
局限与挑战 ：
- 评估规模有限：测试集仅 240 题，虽然难度高，但泛化能力需更大规模验证。
- 幻觉风险：编排器本身不验证事实，可能合成出看似合理但错误的答案，需谨慎对待输出结果。
SciOrch 证明了在资源受限场景下，通过智能编排现有顶级模型，可以有效突破单一模型的性能瓶颈。这种“杠杆式”利用 API 的思路，值得在各类复杂推理任务中深入探索。
