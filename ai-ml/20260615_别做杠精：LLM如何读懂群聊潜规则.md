论文 : LoSoNA: A Benchmark for Local Social Norm Adaptation in Group Conversations链接 : https://arxiv.org/abs/2606.14600现在的 LLM 在单轮对话里是个得体的绅士，但一旦扔进多人群聊，经常因为“不懂事”而社死。
这篇论文提出的 LoSoNA 基准测试，专门考察模型能否从历史聊天记录中，推断出那些没明说的“本地社交规范”，并据此调整回复策略。
对于正在开发 Agent 的工程师来说，这不仅是学术玩具，更是解决“AI 显得太机器人”的关键痛点。
### 为什么现有方案不够用？
现有的社交评估基准（如 Sotopia）大多关注目标达成或双人互动，缺乏对“群体潜规则”的量化测试。
在真实群聊中，规范是隐性的：有的团队讲究高效直接，有的则重视情感共鸣。
如果模型只依赖通用的礼貌原则，它可能会在一个崇尚“极简主义”的技术群里，对同事的 Bug 汇报长篇大论地表示同情，从而显得格格不入。
LoSoNA 的核心直觉在于： 社交能力不是背诵规则，而是从上下文先例中动态推断行为边界。
### 方法拆解：如何制造“社死现场”？
LoSoNA 的设计非常克制且精准，它通过三个步骤剥离了模型的“通用礼貌”伪装：
- 场景生成：构建包含隐藏规范（如“避免道歉”、“简洁回答”）的群聊上下文。
- 无演示筛选：这是最狠的一招。在评估前，先用基线模型测试“无先例”情况。如果基线模型默认就符合规范，该场景被剔除。
Insight：确保只有那些需要“主动适应”而非“默认正确”的场景进入测试集，迫使模型必须观察历史才能得分。
- 单轮评估：固定对话历史，让模型对最后的触发消息（Elicitor）做出一次性回复。
这种设计避免了多轮交互中的噪声，直接测量模型在关键时刻的“察言观色”能力。
### 关键结果：提示词比模型参数更关键？
论文测试了 8 款前沿模型，结果令人惊讶： 简单的 Prompt 工程带来的提升，远超模型本身的基线差异。
在默认（Naive）条件下，大多数模型的表现惨不忍睹，平均准确率仅为 33.2% 。
但当使用 norm_informed 提示词（明确告知存在本地规范）时，整体准确率跃升至 46.1% 。

### 工程启示- Prompt 策略需动态调整：不要对所有 Agent 使用统一的“请遵守社交礼仪”指令。对于复杂群聊场景，显式提示“注意观察群内沟通习惯”能显著提升 Gemini 和 Claude 的表现。
- 模型选型要看“可塑性”：如果你的业务强依赖本地化适配（如企业内部 Slack 机器人），Gemini 3.1 Pro 和 Claude Fable 5 目前展现出更强的规范推断潜力。
- 警惕“过度适应”：对于 Mistral 等模型，简单的规范提示可能导致回归。在实际部署中，需要通过少量样本验证 Prompt 的有效性，避免“好心办坏事”。
### 局限与展望LoSoNA 目前仅包含 38 个经过人工筛选的场景，且均为英语。
它测试的是单轮反应，而非长期的社交修复能力。
未来，随着多规范冲突、跨语言场景的加入，这个基准将更贴近真实的混乱群聊环境。
对于工程师而言，现在的核心任务不是追求更高的参数量，而是让模型学会“看脸色”——毕竟，在群里装傻容易，读懂潜规则难。
