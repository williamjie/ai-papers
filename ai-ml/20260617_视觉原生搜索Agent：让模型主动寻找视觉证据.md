# ⭐⭐½ 视觉原生搜索Agent：让模型主动寻找视觉证据

**日期**: 2026-06-17

---

论文 : Visual-Seeker: Towards Visual-Native Multimodal Agentic Search via Active Visual Reasoning链接 : https://arxiv.org/abs/2606.15231现在的多模态大模型（MLLMs）虽然能看图，但在复杂的开放世界搜索任务中，往往缺乏“事实锚定”能力。蚂蚁集团联合学术界提出的 Visual-Seeker 试图解决这个痛点：它不再把视觉当作静态输入，而是让 Agent 具备“主动视觉推理”能力，在搜索过程中动态收集并整合视觉证据。
### 现有方案的痛点：视觉只是“附加题”
目前的图文搜索 Agent 存在两个核心缺陷：
- 训练数据过于理想化：现有方法多使用语义明确的简单图片构建训练集，导致模型在面对真实世界中背景复杂、实体众多的图片时，细粒度感知能力不足。
- 缺乏主动视觉取证：大多数 Agent 仅将视觉查询插入提示词，搜索轨迹主要依赖文本证据。它们无法在网页中主动寻找并解析图片细节（如颜色、布局），限制了多跳跨模态推理的深度。
### 核心 Insight：从“被动看”到“主动找”
Visual-Seeker 的核心直觉是： 视觉原生能力必须通过“主动收集视觉证据”来训练 。为此，作者设计了一套 Active Visual Reasoning 数据合成流水线，包含三个关键步骤：
-细粒度种子实体选择：
从真实复杂图片中提取实体。不同于传统 NER，这里利用 MLLM 联合视觉和文本线索，提取出如“图中穿粉色衬衫的男人”这样的结构化实体，并通过规则过滤掉模糊指代（如“那个人”），确保训练数据的高难度和高精度。
-双策略随机游走构建多跳轨迹：
基于离线维基百科知识图谱进行随机游走生成推理链。
回溯策略（Backtracking）：模拟人类搜索时的假设修正，以概率 pbackp_{back}​ 跳回历史节点，形成树状结构，迫使 Agent 管理多条推理分支。
- 循环约束（Cycle Constraint）：强制生成包含分叉和汇合的环形拓扑，增加路径多样性。
-视觉证据注入（Visual Evidence Injection, VEI）：
这是最关键的创新。对于每个问答对，系统检索相关图片，利用 MLLM 提取仅凭文字无法推导的细粒度视觉细节（如“羽毛的颜色”）。这些细节被合成新的两跳 QA 对并合并入原始轨迹，强制模型调用 search_image 工具进行像素级推理。
### 关键结果：SFT 即可超越闭源模型Visual-Seeker 基于 Qwen3-VL-8B-Instruct，仅使用 5K 条 高质量合成轨迹进行监督微调（SFT），未使用昂贵的强化学习。
在五个极具挑战性的基准测试中，其表现令人印象深刻：

消融实验揭示了关键依赖：
- 数据层面：引入“含视觉证据”的数据后，VisBrowse 准确率从 20.1% 飙升至 34.7%。
- 工具层面：移除 image_crop 或 search_image 工具会导致性能显著下降（平均下降 5.6%-8.9%），证明细粒度裁剪和主动图像搜索是视觉原生能力的基石。
### 工程启示- 数据质量 > 数据数量：仅用 5K 条精心合成的轨迹，通过 SFT 就能让开源小模型在特定任务上超越千亿参数闭源模型。这提示我们，针对 Agent 场景，构建包含“主动工具调用”和“多模态证据链”的高质量合成数据比盲目堆砌数据更有效。
- 视觉工具链的必要性：单纯的图文理解不足以支撑复杂搜索。工程中必须集成 image_crop（细粒度定位）和 search_image（反向/正向搜图）工具，并让模型学会在何时调用它们。
- ReAct 模式的视觉扩展：Visual-Seeker 证明了 ReAct 范式可以自然扩展到多模态领域。关键在于 Prompt 中不仅要包含文本推理，还要显式引导模型关注网页中的视觉证据。
### 局限与展望虽然 Visual-Seeker 在基准测试上表现优异，但其依赖离线维基百科知识图谱进行轨迹合成，可能限制了其在实时性要求极高或长尾知识领域的泛化能力。此外，目前主要基于 SFT，未来结合强化学习（RL）可能会进一步提升其在开放网页环境中的鲁棒性。
对于工程师而言，这篇论文提供了一个清晰的范式： 不要只教模型“看图”，要教模型“为了回答问题去主动找图” 。
