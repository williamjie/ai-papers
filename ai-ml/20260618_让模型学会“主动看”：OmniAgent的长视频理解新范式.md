# ⭐⭐⭐ 让模型学会“主动看”：OmniAgent的长视频理解新范式

**日期**: 2026-06-18

---

论文 : Native Active Perception as Reasoning for Omni-Modal Understanding链接 : https://arxiv.org/abs/2606.19341处理长达数小时的视频时，现有大模型通常采用“一视同仁”的被动策略：要么均匀采样帧，要么全量输入。这种“看全貌”的做法导致计算成本随视频时长呈超线性增长，且容易在海量冗余信息中淹没关键线索。阿里通义千问团队提出的 OmniAgent 彻底改变了这一逻辑，它将视频理解重构为部分可观测马尔可夫决策过程（POMDP），让模型像人类一样“主动感知”。
核心直觉：把“看”变成一种推理行为。
传统多模态大语言模型（MLLM）是被动接收者，而 OmniAgent 将感知动作内化为推理循环的一部分。它不一次性处理所有画面，而是通过迭代式的观察-思考-行动（Observation-Thought-Action, OTA）周期，根据当前查询动态决定“下一步看哪里”或“听哪段音频”。这种设计的关键在于 严格的信息蒸馏 ：模型将高维的瞬时视听感知转化为紧凑的文本记忆，随后立即丢弃原始媒体数据。这意味着模型的上下文负担不再取决于视频时长，而仅由推理链路的复杂度决定。
为了实现这种原生主动感知，OmniAgent 采用了两阶段优化策略。首先是代理监督微调（Agentic SFT），通过最佳 N 轨迹合成与双重质量控制，让模型学会如何执行探索动作并纠正错误。其次是代理强化学习（Agentic RL），引入了 TAURA（Turn-aware Adaptive Uncertainty Rescaled Advantage）算法。
⚠️ 反直觉发现：标准强化学习在长程推理中会“偷懒”
论文指出，直接将 Group Relative Policy Optimization (GRPO) 应用于多轮代理推理时，会出现“优势同质化”问题。即模型将整条轨迹的奖励均匀分配给每一步，导致关键的探索决策与琐碎的动作获得相同的梯度更新。TAURA 利用每轮的 Token 熵作为权重，精准地将信用分配给那些高不确定性的关键转折点，从而强化真正的“发现”行为。
实验结果极具说服力。在长视频理解基准 LVBench 上，7B 参数的 OmniAgent 取得了 50.5% 的准确率，不仅大幅超越基线 Qwen2.5-Omni（43.0%），甚至击败了参数量为其 10 倍的 Qwen2.5-VL-72B（47.3%）。更令人印象深刻的是效率：OmniAgent 平均仅需处理 203 帧 ，而 72B 模型需要密集输入 768 帧。

工程启示：小模型 + 主动策略 > 大模型 + 被动堆料。
对于需要处理长视频或复杂多模态数据的工程师而言，这篇论文提供了一个重要方向：与其盲目增加模型参数量或上下文窗口，不如赋予模型“主动筛选信息”的能力。通过构建 OTA 循环和文本记忆机制，我们可以用较小的模型实现更高效的长程推理。同时，TAURA 算法为解决多步 Agent 训练中的信用分配难题提供了实用的解决方案。
当然，该方法的局限性在于串行交互带来的延迟开销。未来若能在保持主动感知优势的同时引入并行探索机制，其工程落地价值将进一步放大。对于本地部署或实时性要求不极高的场景，OmniAgent 架构值得深入借鉴。
