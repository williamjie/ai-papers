# ⭐⭐⭐½ 视频嵌入基准MVEB：音频是助攻还是噪音？

**日期**: 2026-06-16

---

论文 : MVEB: Massive Video Embedding Benchmark链接 : https://arxiv.org/abs/2606.14958做多模态检索或 Agent 开发的同学，最近一定被各种“全能”视频模型刷屏。但当你真要把它们塞进生产环境时，往往会发现一个尴尬现象：在动作识别上刷榜的模型，搞跨模态检索时直接拉胯。这种“偏科”导致我们很难判断一个模型的 Embedding 质量到底如何。MVEB 的出现，就是为了给视频嵌入领域立一把统一的尺子，并揭示了一个反直觉的工程真相： 音频并不总是视频的加分项。
### 现有方案的痛点：碎片化与评估缺失过去的视频基准测试（如 VideoEval, UVRB）通常只覆盖单一能力，要么是纯视觉分类，要么是单纯的检索。这导致模型开发者容易过拟合特定任务，缺乏通用表征能力的衡量标准。更关键的是，以往基准很少系统性地对比“纯视频”与“视频+音频”的效果差异。在 MVEB 之前，我们甚至不知道加入音频轨道到底会让模型性能提升还是下降，因为这取决于数据标注的来源。
### 方法拆解：MTEB 生态的视频延伸MVEB 并非从零造轮子，而是继承了文本（MTEB）、图像（MIEB）和音频（MAEB）的成熟评估体系。其核心设计直觉在于**“成对评估” 与 “冗余剪枝”**。
- 成对模态评估：对于所有包含音频的数据集，MVEB 强制要求模型运行两次：一次仅输入视频，一次同时输入视频和音频。这种设计允许我们在同一组 clip 上直接测量音频的贡献度（Δ=Scoreva−Scorev\Delta = Score_{va} - Score_vScoreva​−Scorev​），从而剥离出音频信号的真实价值。
- 从 184 到 23 的剪枝策略：全量评估 184 个任务（MVEB+）成本极高。作者通过计算模型排名的斯皮尔曼相关性（Spearman ρ\rho），剔除冗余任务，最终选出 23 个高多样性任务组成 MVEB。实验证明，这种剪枝使评估速度提升了 7-10 倍，同时保持了极高的排名相关性（Pearson r=0.996r=0.9960.996）。
### 关键结果：没有全能王，音频看“出身”
MVEB 评估了 33 个模型，涵盖 MLLM 嵌入、多模态绑定等六大范式。结论非常明确： 没有任何单一模型在所有任务上占据统治地位。
- MLLM-based Embedding（如 LCO-Embedding-Omni-7B）在分类、聚类和问答任务上领先。
- Multimodal Binding（如 eBind）在检索和零样本分类上表现更佳。
- 纯生成式 MLLM（如 Qwen2.5-Omni 直接取隐藏层）在跨模态任务上彻底崩盘，得分仅为经过对比适应模型的十分之一。
⚠️ 反直觉发现：音频贡献取决于标注来源论文揭示了一个极具工程指导意义的结论：音频是否有帮助，完全取决于数据集的标签是如何产生的。
- AV-grounded 数据集（标签由音视频共同产生）：音频带来 +0.016 的平均增益。
- V-grounded 数据集（标签仅由视觉产生）：音频造成 -0.046 的性能下降。
这意味着，如果下游任务的 Ground Truth 是纯视觉定义的，强行引入音频反而会成为噪声干扰，导致性能显著劣化。
### 工程启示：选型与调优指南对于正在构建视频理解应用的工程师，MVEB 提供了三条务实建议：
- 对比学习是刚需：不要试图直接用生成式 MLLM 的隐藏层做检索。数据表明，经过专门的对比学习（Contrastive Learning）适应阶段，模型性能会有数量级的提升。e5-omni-7B (55.0分) 远超其基座 Qwen2.5-Omni-7B (12.8分)。
- 按需裁剪模态：如果你的业务场景主要是基于视觉标签的检索（如电商视频搜索），使用纯文本-视频编码器（如 Qwen3-VL-Embedding）可能比全模态模型效果更好，且推理成本更低。MVEB(text, video) 榜单显示，无音频路径的模型在此类任务上甚至击败了全模态巨头。
- 帧数预算的边际效应：测试表明，从单帧增加到 8 帧带来 43.7% 的性能跃升，但从 32 帧增加到 64 帧仅提升 2.2%。在生产环境中，将采样帧数限制在 32 帧是性价比最高的选择。
### 局限与展望MVEB 目前仍存在一些覆盖盲区。长视频内容（如讲座、纪录片）因计算成本高昂而缺失；低资源语言和特定垂直领域（如医疗、手语）的数据集较少。此外，由于许多模型训练数据未公开细节，部分评估结果可能存在训练-测试污染的风险。尽管如此，MVEB 提供的标准化协议和开源代码，已经为视频嵌入模型的公平比较提供了最佳实践基线。
