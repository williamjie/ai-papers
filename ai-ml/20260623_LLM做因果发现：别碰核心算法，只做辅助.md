# ⭐⭐⭐⭐ LLM做因果发现：别碰核心算法，只做辅助

**日期**: 2026-06-23

---

论文 : Causal Discovery in the Era of Agents链接 : https://arxiv.org/abs/2606.23608最近很多团队试图把大语言模型（LLM）塞进因果发现（Causal Discovery）的流程里。有的让 LLM 直接猜边方向，有的把它当先验知识注入搜索算法。这篇来自 CMU 和 MBZUAI 的论文泼了一盆冷水： 别这么干 。
作者提出了一个反直觉但极具工程价值的观点： Agent 应该辅助工作流，但绝不能提供因果证据。
### 为什么现有方案很危险？
目前的趋势是“让 LLM 参与推理”。但这带来了一个致命问题： 混淆来源 。
当你把 LLM 的输出作为边、方向或约束注入图构建过程时，你无法区分最终发现的因果关系是来自：
- 数据中的统计证据- 预训练语料中的背景知识- Prompt 的措辞偏差- 或者是单纯的幻觉（Hallucination）
因果发现的核心价值在于 可解释性 和 假设明确性 。如果图里的箭头是 LLM “猜”出来的，你就失去了对证据链的控制。一旦这些来源混合，图的证据状态就变得模糊不清。
⚠️ 核心 Insight ：LLM 擅长预测文本模式，不擅长提供基于数据的因果证据。它的“流利解释”往往会让部分识别的图对象（如 CPDAG）听起来比实际更确定，导致过度解读。
### 方法拆解：causal-learn+ 的设计哲学作者没有提出新的数学算法，而是重新定义了 Agent 在因果发现中的 边界 。他们构建了 causal-learn+ 平台，基于开源库 causal-learn ，实施了一套严格的“隔离原则”。
Agent 的角色被限定为 工作流助手 ，具体包括五个方面，但必须遵守以下红线：
- 数据理解：总结变量、检测缺失值、识别异常分布。
红线：不能自动决定丢弃哪些变量。
- 预处理建议：解释标准化、离散化对图的影响。
红线：所有预处理决策必须由用户显式批准，Agent 不能静默执行。
- 算法推荐：根据数据特征推荐约束基（Constraint-based）、评分基（Score-based）或潜变量方法。
红线：Agent 不选择最终算法，只解释假设差异。
- 专家知识检索：通过 RAG 提供领域背景。
红线：领域知识不能直接变成图中的边或约束，除非用户手动采纳。
- 结果解读：解释 CPDAG、PAG 等符号含义，避免过度解读。
核心设计直觉 ：将“辅助”与“证据”分离。Agent 处理的是上下文、指导和解释；而条件独立性检验、评分、秩约束、图搜索和方向规则，必须由形式化算法在受保护的核心中计算。
### 关键结果：大五人格数据案例论文使用包含近 20,000 份完整在线回复的大五人格（Big Five）问卷数据集进行了案例研究。该数据集包含 50 个观测指标，每个维度 10 个问题。
由于涉及潜变量（如外向性、神经质等无法直接测量的特质），作者使用了 RLCD （处理因果相关隐藏变量的框架）结合 GIN （用于定向隐藏变量间的关系）。
- 发现：算法成功将属于同一维度的观测指标聚类到共同的隐藏变量附近。
- 新假设：揭示了如 L1→L6→L3L1 \rightarrow L6 \rightarrow L3L6→L3 等潜在路径，以及 O2→O4O2 \rightarrow O4O4 等观测链接。
- Agent 的作用：在预处理阶段解释为何需要标准化；在解读阶段指出隐藏节点可能对应“尽责性”，但强调这只是假设而非确证事实。
这个案例证明了： 即使在没有 LLM 参与核心推理的情况下，Agent 也能显著降低使用复杂因果发现工具（如处理潜变量）的门槛。
### 工程启示对于正在构建 AI Agent 系统的工程师，这篇论文提供了清晰的架构指导：
- 信任边界：在科学计算、金融风控等高风险领域，LLM 应作为“UI 增强层”或“文档检索层”，而非“决策引擎”。
- 可追溯性（Traceability）：系统必须记录 Agent 的建议、用户的确认、算法的参数和最终输出。如果无法区分哪些是 LLM 生成的，哪些是算法计算的，这个系统就是不可信的。
- 避免静默操作：Agent 不应自动执行数据清洗或参数调优。所有影响因果结论的操作必须是可见的、可逆的，并需用户审批。
### 局限与展望论文承认，当前的 LLM 缺乏形式化的因果保证。未来的方向包括：
- 开发基准测试，评估 Agent 在数据诊断和方法选择上的准确性。
- 构建审计追踪（Audit Trails），严格分离数据、假设和算法输出。
- 长期来看，可能需要训练具有明确假设和可识别性条件的“因果基础模型”，但在这一天到来之前，“Agent 辅助，算法发现，科学家判断” 是唯一安全的设计范式。
