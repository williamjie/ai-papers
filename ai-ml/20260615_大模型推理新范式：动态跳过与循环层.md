论文 : Skip a Layer or Loop It? Learning Program-of-Layers in LLMs链接 : https://arxiv.org/abs/2606.06574大模型推理一直是个“死脑筋”活计：无论输入是简单问候还是复杂数学题，它都老老实实从头跑到尾。这篇论文提出了一个大胆的想法：把预训练层看作可复用的函数库，让模型自己决定“跳过”还是“循环”。这不仅提升了准确率，还常常能减少计算量。
### 为什么固定深度不够用？
现有动态推理方案（如 Early Exit）通常只允许“提前退出”，或者像 Loop Transformer 那样重复同一层。但这太局限了。论文通过蒙特卡洛树搜索（MCTS）发现，预训练模型内部存在大量“潜在有效程序”。
核心 Insight ：标准前向传播只是众多正确推理路径中的一种，且往往是“过度计算”的那一种。
### 方法拆解：PoLar 的三段式设计作者提出了 PoLar (Program-of-Layers) ，用一个轻量级预测网络替代昂贵的在线搜索。其设计直觉非常精妙：
- 模块化分割：不针对单层做决策，而是将连续层打包成“段”。实验发现，有效程序主要由长度不超过 4 层的连续段组成（54.5% 的段仅含 1 层）。
- 极简操作集：每个段只有三种命运——skip（跳过）、keep（保留）、repeat（重复一次）。MCTS 数据显示，绝大多数有效程序不需要多次重复同一模块。
- 全局感知预测器：使用一个仅含 210 万参数的轻量网络，通过 Cross-Attention 同时关注输入 Token 和所有层索引，实现端到端的程序生成。
### 关键结果：更准、更快、更强在 DART-Math 数学推理基准上，PoLar 的表现令人印象深刻。以 LLaMA-3.2-3B 为例：
难度等级 Base (标准) PoLar (Pass@1) 提升幅度 DM-1 (易) 40.6% 46.2% +5.6% DM-3 (中) 27.6% 30.2% +2.6% DM-5 (难) 28.6% 30.2% +1.6%更惊人的是 Pass@k 的测试时缩放能力。当允许生成多个候选程序时，PoLar 在 DM-1 上从 46.2% 飙升至 68.4% ，远超随机采样的基线（47.6%）。
⚠️ 反直觉发现 ：对于原本就答对的样本，75.5% 存在更短的等效程序；对于原本答错的样本，36.2% 也能通过更少层数修正。这意味着标准推理经常在做无用功。
在延迟方面，PoLar 的额外开销仅相当于 0.23 个 LLM 层 （约 3ms）。在简单任务上，端到端延迟降低至基线的 0.83 倍 。
### 工程启示- 零成本部署：PoLar 不需要微调主模型参数，只需训练一个极小的预测头。这对于冻结权重的商业 API 或本地大模型极具吸引力。
- 超越 Early Exit：单纯的“跳过”只能提速，而“循环+跳过”的组合能显著提升复杂推理能力。如果你的业务场景包含高难度任务，值得尝试这种混合策略。
- 泛化能力：在 MMLU-Pro 等分布外数据集上，PoLar 依然稳定优于基线，说明它学到的是通用的计算控制策略，而非特定数据的过拟合。
### 局限与展望目前 PoLar 主要验证于数学推理任务，且在极难任务上的提升幅度相对较小（DM-5 仅 +1.6%）。此外，预测网络的准确性依赖于 MCTS 生成的标签质量，若搜索空间覆盖不足，可能限制上限。未来若能扩展操作集（如支持多次重复），或许能进一步挖掘模型的深层潜力。
