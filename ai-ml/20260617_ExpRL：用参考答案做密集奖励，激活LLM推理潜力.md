# ⭐⭐⭐½ ExpRL：用参考答案做密集奖励，激活LLM推理潜力

**日期**: 2026-06-17

---

论文 : ExpRL: Exploratory RL for LLM Mid-Training链接 : https://arxiv.org/abs/2606.17024大模型强化学习（Reinforcement Learning, RL）的痛点不在于算法本身，而在于“冷启动”。如果基础模型根本不知道解题的大致方向，稀疏的最终答案奖励（Sparse Reward）就无法提供有效梯度。Stanford 和 OpenAI 联合发表的 ExpRL 提出了一种全新的 Mid-Training 思路：不模仿参考答案，而是将其作为“评分标准”，生成密集奖励来引导模型探索。
### 现有方案的死胡同目前主流的 RL 预处理手段主要有两类，但都有明显缺陷：
- 监督微调（SFT）：直接让模型模仿人类写的解题步骤。问题是，如果基础模型完全不具备这种推理能力，强行模仿会导致灾难性的性能下降。
- 稀疏奖励 GRPO：直接用最终答案对错作为奖励。在难题上，模型可能 64 次尝试全错，导致训练早期完全没有信号，陷入死循环。
ExpRL 的核心洞察是： 模型虽然不会解题，但擅长验证 。利用这种“验证-生成差距”，我们可以把参考答案变成一把尺子，而不是一个靶子。
### ExpRL：从模仿到探索ExpRL 的设计非常巧妙，它不向策略网络（Policy）暴露参考答案的具体内容，而是让一个裁判模型（LLM Judge）拿着参考答案和模型生成的推理过程进行对比，给出密集分数。
具体分为两种奖励机制：
- ExpRL-Outcome：对完整的推理轨迹打分。即使最终答案错了，只要中间步骤与参考答案的逻辑高度一致，就能获得高分。这解决了稀疏奖励“非黑即白”的问题。
- ExpRL-Process：更激进的做法。它将推理过程切分成多个前缀片段（Prefixes），对每个片段都进行打分。通过计算相邻片段的分数差（Advantage），模型能明确知道哪一步推进了进度，哪一步在倒退。
⚠️ 关键直觉 ：ExpRL 的目标不是让模型记住参考答案，而是通过密集奖励，增加模型采样到“有效推理路径”的概率质量。这直接提升了 pass@k 指标，为后续的稀疏奖励 RL 铺平了道路。

- SFT 的失败：在 AIME-25 上，SFT 后的模型性能甚至不如基础模型（从 46.46% 暴跌至 26.62%），验证了盲目模仿高阶推理路径的危害。

### 工程启示与局限对于工程师而言，ExpRL 提供了处理高难度推理任务的新范式：
- 不要迷信 SFT：当基础模型能力不足时，SFT 可能是毒药。利用 RL 进行探索性训练是更安全的选择。
- 小裁判大模型可行：论文实验显示，使用较小的 Qwen3-4B 作为裁判，为较大的 Qwen3-8B 提供密集奖励，依然能在数学和科学领域带来显著提升。这意味着你可以用低成本的小模型来指导大模型的训练。
当然，ExpRL 也有边界。在代码生成任务（LiveCodeBench）上，由于代码执行的稀疏奖励本身非常强且客观，ExpRL 的优势不明显，甚至略逊于标准的 Sparse GRPO。这说明在拥有强环境反馈的领域，复杂的参考系评分可能并非最优解。
总之，ExpRL 证明了将“参考答案”转化为“密集反馈信号”的价值。它让模型在学会解题之前，先学会了如何正确地思考。
