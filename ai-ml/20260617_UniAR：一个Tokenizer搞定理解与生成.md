# ⭐⭐⭐ UniAR：一个Tokenizer搞定理解与生成

**日期**: 2026-06-17

---

论文 : Unified Multimodal Autoregressive Modeling with Shared Context-Visual Tokenizer is Key to Unification链接 : https://arxiv.org/abs/2606.18249现有的统一多模态模型（Unified Multimodal Models）大多存在一个尴尬的“精神分裂”：理解和生成使用两套完全不同的视觉分词器。这导致模型生成的图像必须重新编码才能被自己理解，彻底破坏了“共享上下文”的初衷。阿里通义千问团队提出的 UniAR 试图通过一个离散的视觉分词器解决这个问题，让模型在同一个语义空间内完成看图、生图和修图。
### 为什么现有方案不够好？
目前的统一建模主要面临两个痛点：
- 表征割裂：理解任务需要高层语义（如“一只猫”），而生成任务依赖底层细节（如毛发纹理）。大多数工作（如 BAGEL, Janus-Pro）被迫使用双 Tokenizer，导致生成的图像无法直接在当前对话中被解析。
- 效率瓶颈：为了保留细节，传统 VQ-VAE 需要巨大的码本或极长的序列，严重拖慢自回归推理速度。
UniAR 的核心 Insight 非常直接： 如果理解和生成共用一套视觉 Token，模型就能像理解文本一样，直接“读懂”自己刚刚生成的图片，无需任何额外的重编码步骤。
### 方法拆解：如何用一套 Token 兼顾语义与细节？
UniAR 的设计并非简单的拼接，而是针对上述痛点做了三个关键改进：
1. 无查找表的比特量化（Lookup-free Bitwise Quantization）
传统 VQ-VAE 依赖显式码本，扩展词汇量成本高。UniAR 采用二进制球面量化（BSQ），将视觉特征映射为 64 位的二进制向量。
- 优势：无需维护巨大的码本，理论词汇量达到 2642^{64}，以极小的计算开销实现了海量视觉表达的覆盖。
2. 多级特征融合（Multi-level Feature Fusion）
为了解决“语义 vs 细节”的矛盾，UniAR 不仅使用 ViT 的最深层特征，还融合了三个中间层的特征。
- 直觉：浅层保留高频细节（纹理、颜色），深层捕捉高层语义（物体类别）。这种混合输入让单个 Token 既懂“是什么”，又知“长啥样”。
3. 并行比特预测与分辨率上采样为了加速生成，UniAR 引入了并行比特预测机制。模型不是逐个像素生成，而是每次预测一个 2×22\times2 2 空间网格内的多级比特向量。
- 效果：实现了 32x 的视觉序列压缩率。对于 1024×10241024\times10241024 的图片，仅需预测 256 个视觉 Token，大幅降低了自回归步数。

### 工程启示与局限对实际应用的指导意义：
- Agent 场景的天然适配：由于 UniAR 能在同一上下文中直接理解自己生成的图片，它非常适合多轮交互的 Agent 应用（如：先生成草图，再根据草图回答细节问题），无需额外的视觉编码开销。
- RLHF 的可行性：论文展示了通过强化学习（Reinforcement Learning）进一步提升生成质量的路径。在 OneIG-EN 上，经过 500 步 RL 训练后，得分从 71.1 提升至 84.0，证明了离散视觉 Token 适合进行奖励模型优化。
局限与展望：
- 推理延迟：虽然序列长度压缩了 32 倍，但自回归生成依然比扩散模型慢。并行比特预测缓解了这一问题，但在高并发场景下仍需关注吞吐量。
- 纯文本能力：在 MMMU 等需要强逻辑推理的基准上，UniAR (44.3) 落后于专用 VLM (Qwen3-VL: 69.6)。这是因为预训练数据中缺乏纯文本语料，且未对理解任务进行 RL 优化。
UniAR 证明了“一个 Tokenizer 走天下”在技术上是可行的，且在生成质量和指令遵循上达到了新高度。对于追求端到端统一建模的工程团队来说，其多级特征融合和比特量化思路值得深入借鉴。
