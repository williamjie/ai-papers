**日期**: 2026-06-19

论文 : HiLo-Token: Input-Adaptive High-Low Frequency Token Compression for Efficient Image Editing链接 : https://arxiv.org/abs/2606.13898在 Adobe Photoshop 的生成式填充（Generative Fill）中，DiT 模块占据了 73% 的推理延迟。这篇论文提出了一种极简的 Token 压缩方案 HiLo-Token，通过区分高低频信息，在不损失画质的前提下实现了最高 3.13 倍的加速。
### 痛点：DiT 太贵，且“一刀切”浪费算力从 U-Net 转向 Diffusion Transformer (DiT) 后，虽然生成质量提升，但推理成本激增近 6 倍。即使经过 8 步蒸馏（Distillation），DiT 依然是瓶颈。
现有的 Token 压缩方法大多基于 Attention 分数或学习重要性预测。但在图像编辑场景中，这有两个致命缺陷：
- 早期无效：扩散过程初期，内容尚未生成，Attention 信号缺失。
- 遮挡失效：例如移除遮挡教堂的交通标志，被遮挡部分在早期无 Attention 响应，导致关键上下文丢失。
### 核心 Insight：高低频分离，各取所需HiLo-Token 的核心直觉非常工程化： 编辑区域保留细节，背景区域只需结构。
作者将 Token 分为两类处理：
- 低频全局结构：将图像下采样 16 倍，仅用极少量 Token 保留模糊的全局轮廓。这部分计算量几乎可以忽略。
- 高频局部细节：这是重点。对于非编辑区域，使用 Sobel 算子提取空间频率图（Spatial Frequency Map）。
⚠️ 反直觉设计 ：作者抛弃了复杂的 Attention 预测，仅用两个卷积操作和一次线性层计算边缘/频率图。这仅需约 10ms 的额外开销，却比基于语义的预测更鲁棒。
具体流程如下：
- 编辑区：保留膨胀后的 Mask 内所有 Token，确保局部连贯性。
- 非编辑区：通过 Sobel 算子筛选高频 Token（如纹理、边缘），低频部分由下采样分支补充。
- 区域化聚合：为了避免 Token 过于分散，对频率图进行 16x 空间池化，形成连贯的区域级选择。
### 关键结果：速度起飞，画质无损在 A100-80GB GPU 上，针对不同 Mask 比例的编辑任务，HiLo-Token 带来了显著的 DiT 加速：
Mask 比例类别 平均占比 DiT 加速倍数 端到端加速 小 Mask 6.38% 3.13x 1.77x 中 Mask 15.92% 2.59x 1.66x 大 Mask 35.36% 1.67x 1.33x在用户偏好测试中，HiLo-Token 与未压缩模型的“平局”率高达 48%-81%，且在许多情况下甚至略胜一筹。这意味着 压缩并未引入肉眼可见的质量下降 。
此外，该方法可与 FP8 量化结合，进一步降低 40% 延迟；也可配合 5 步蒸馏，在可接受画质损失下再提速 37.5%。
### 工程启示：简单往往最有效这篇论文对工业界落地有两个重要指导意义：
- 不要过度依赖 Attention 做剪枝：在生成式任务中，Attention 信号具有滞后性和不稳定性。基于物理属性（如频率、梯度）的启发式方法，计算成本更低且更稳定。
- 输入自适应优于模型压缩：与其静态地剪枝模型层数，不如根据输入 Mask 的动态分布分配 Token 预算。对于小面积编辑，保留高频细节比保留大面积低频背景更重要。
### 局限与展望该方法目前主要针对基于 Mask 的图像编辑任务。对于无 Mask 的全图生成，低频下采样可能丢失关键语义信息。此外，Sobel 算子对某些平滑渐变区域的效果可能不如学习到的特征敏感，未来可探索轻量级的学习型频率预测器。
