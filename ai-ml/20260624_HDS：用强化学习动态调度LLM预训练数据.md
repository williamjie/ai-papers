论文 : Holistic Data Scheduler for LLM Pre-training via Multi-Objective Reinforcement Learning链接 : https://arxiv.org/abs/2606.24133大模型预训练正在从“算力驱动”转向“数据驱动”。静态混合策略（Static Mixing）假设一个固定配方贯穿全程，这显然违背了模型学习的非平稳特性。这篇论文提出的 HDS（Holistic Data Scheduler）试图用强化学习解决这个动态调度难题，核心亮点在于它不仅仅看 Loss，而是构建了一个多维度的奖励函数。
### 为什么现有方案不够好？
现有的在线数据混合（Online Data Mixing, ODM）方法主要存在两个痛点：
- 视野狭窄：早期的 ODM [2] 使用多臂老虎机模型，仅以域损失作为奖励，忽略了域之间的协同效应。
- 缺乏全局观：后续如 AC-ODM [16] 引入了梯度对齐，但依然只关注单一视角（如数据质量或训练动态），未能综合考虑模型稳定性与课程学习节奏。
HDS 的核心 Insight 是： 预训练是一个多目标优化过程，需要同时兼顾数据质量、域间影响和模型稳定性。
### 方法拆解：多维奖励函数设计HDS 将数据调度建模为马尔可夫决策过程（MDP），采用 Soft Actor-Critic (SAC) 算法进行连续控制。其核心创新在于设计了三个维度的奖励信号：
-域间影响奖励 (ralignr_{align}​)
直觉：如果一个域的数据更新方向与其他域一致，说明它有助于全局收敛。
- 实现：计算当前域梯度与其他所有域梯度的内积和。高内积意味着该域数据具有“正外部性”，能加速整体学习。
-调度词汇多样性奖励 (rdiversityr_{diversity}​)
直觉：模型早期应学习简单文本，后期逐步接触复杂文本（课程学习）。
- 实现：使用 MTLD 指标衡量文本复杂度。奖励函数随训练步数动态调整：早期鼓励低 MTLD，晚期鼓励高 MTLD。这相当于自动构建了从易到难的数据流。
-模型稳定性奖励 (rstabilityr_{stability}​)
直觉：剧烈的权重波动往往意味着过拟合或震荡，不利于泛化。
- 实现：基于选定层权重的 L2 范数变化率。变化越小，奖励越高。这充当了正则化角色，防止训练过程过于激进。
⚠️ 关键设计细节 ：Actor 和 Critic 网络采用了紧凑的 Transformer 架构（约 500 万参数），既能处理高维状态向量，又不会成为计算瓶颈。
### 实验结果：效率与性能双提升在 Pythia-1B 模型、The Pile 数据集上的实验显示，HDS 显著优于静态基线和其他动态方法：

- 性能增益：在 MMLU 0-shot 任务上，HDS 相比 AC-ODM 提升了 7.2%（从 0.251 到 0.269）。
消融实验进一步证实了各奖励项的作用：移除 ralignr_{align} ​ 导致性能大幅下降，证明 域间梯度对齐是加速收敛的核心驱动力 ；移除 rdiversityr_{diversity} ​ 则延缓了早期收敛速度，验证了课程学习的有效性。
### 工程启示- RL 调度的可行性：SAC 算法在连续动作空间上的稳定性使其适合此类高维调度任务。虽然引入了 RL Agent，但单步计算开销仅增加 <1%（2.47s -> 2.49s），性价比极高。
- 课程学习的自动化：传统课程学习需要人工定义难度曲线，HDS 通过 rdiversityr_{diversity}​ 自动适应模型当前能力，减少了超参调试成本。
- 扩展性：Agent 参数仅占 LLM 的 ~0.5%，随着模型规模扩大（如 7B+），相对开销可忽略不计，适合大规模预训练场景。
### 局限与展望- 状态空间依赖：HDS 的状态向量包含权重范数等信息，这在分布式训练中可能需要额外的通信开销来同步全局统计量。
- 奖励权重敏感：论文中 walign=1,wdiversity=10,wstability=10w_{align}=1, w_{diversity}=10, w_{stability}=10​=1,wdiversity​=10,wstability​=10 的配比是通过实验调优得到的，不同数据集或模型架构可能需要重新校准这些系数。
HDS 为动态数据混合提供了一个更全面的视角，证明了通过多目标强化学习可以同时优化训练速度和最终模型能力。对于追求极致预训练效率的团队来说，这是一个值得深入探索的方向。
