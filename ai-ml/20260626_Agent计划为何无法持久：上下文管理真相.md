# ⭐⭐⭐ Agent 计划为何无法持久：上下文管理真相

**日期**: 2026-06-26

---

论文 : Plans Don’t Persist: Why Context Management Is Load Bearing for LLM Agents链接 : https://arxiv.org/abs/2606.22953做长程 Agent 的同学都知道，为了控制成本，我们习惯对上下文进行压缩或剔除。但一个致命盲区是：你删掉的 token，模型真的“记住了”吗？这篇来自 Snowflake AI Research 的论文给出了反直觉答案： LLM 根本没记住计划 。
### 为什么这很重要现有的 ReAct、CoT 等框架默认 Agent 会内化（Internalize）计划。这意味着即使计划文本被移出窗口，Agent 依然能靠内部状态执行。如果这个假设不成立，任何基于“压缩旧上下文”的优化策略都可能导致 Agent 在长任务中突然“失忆”。
### 核心方法：重放配对诊断作者设计了一个巧妙的实验—— 重放配对（Replay Pairing） 。
针对同一个任务轨迹，跑两遍：
- 条件 A：正常保留计划文本在历史中。
- 条件 B：剔除计划文本，但强制输入与 A 完全相同的动作和观察结果。
通过计算两步之间隐藏状态（Hidden State）的余弦距离，量化“计划信号”的存在感。如果模型真的记住了计划，移除文本后隐藏状态不应有显著差异；反之，则说明模型只是在“重读”上下文。
### 关键发现：计划信号极速衰减在 Llama-3.1-70B 上，结果令人震惊。计划注入后的第一步（Step+1），计划信号峰值高达 0.453 。然而，仅仅经过一个动作-观察循环后，该信号暴跌 4.1 倍 至 0.110。到第 5 步时，信号几乎归零（0.027）。
在 HotpotQA 数据集上，这种衰减更剧烈，单步跌幅达 12.4 倍 。
⚠️ 反直觉结论 ：LLM 并没有将计划转化为持久的内部状态。它极度依赖计划文本留在上下文中。一旦滑动窗口移出计划，模型就“忘”了要做什么。
### 推理模型的“幻觉”与修正在测试 DeepSeek-R1 等推理模型时，作者发现标准方法下的信号极弱（0.022 vs Llama 的 0.453）。这并非因为 R1 更擅长记忆，而是因为 推理轨迹污染（Reasoning-trace Confound） 。
R1 在每一步的 <think> 块中会重新推导计划内容。如果只删除原始计划文本，模型依然能从历史思考痕迹中读到计划，导致测量失真。作者提出了 严格剥离（Strict Stripping） ：不仅删计划，还删除历史中的 <think> 块。
修正后，R1 的计划信号回升 +163% ，证明推理模型同样依赖上下文重读，而非内部存储。
### 工程启示与代价论文最后进行了残酷的压力测试：在 ALFWorld 环境中强制剔除计划文本。
- 朴素压缩：任务成功率暴跌 34.7%（从 56.7% 降至 22.0%）。
- 探针恢复：即使用探针检测衰减并重新注入计划，也无法挽回损失。
这是因为在极短的上下文预算下（仅保留最近 4 条消息），被挤出的不仅是过时的计划，更是 Agent 当前需要的“工作状态”。
### 总结这篇论文给所有试图优化长程 Agent 成本的工程师泼了一盆冷水： 不要假设模型会内化指令。
如果你的系统依赖上下文压缩，必须确保关键信息（如核心计划、安全约束）始终位于窗口内，或者通过微调让模型真正学会“记忆”而非“重读”。目前看来，后者远未成熟。
