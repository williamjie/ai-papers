# ⭐⭐⭐⭐ 用强化学习动态混数据，训练提速1.46倍

**日期**: 2026-06-23

---

论文 : AC-ODM: Actor—Critic Online Data Mixing for Sample-Efficient LLM Pretraining链接 : https://arxiv.org/abs/2505.23878大模型预训练中，数据配比（Data Mixing）往往决定了模型的最终上限。
传统的静态混合策略在训练初期可能有效，但随着模型能力的演进，固定的权重无法适应动态的学习过程。
这篇来自 ICML 2026 的论文 AC-ODM 提出了一种全新的思路：把数据混合看作一个强化学习（Reinforcement Learning, RL）问题，让 Agent 实时感知模型状态并调整采样策略。
为什么值得关注？
因为它不仅提升了收敛速度，更关键的是，它在几乎不增加计算开销的前提下，实现了显著的性能跃升。对于算力昂贵的工程实践而言，这种“零成本”的效率提升极具吸引力。
### 痛点：静态混合的局限与动态方法的笨重现有的数据混合方法主要分为两类：静态策略（如 DoReMi、CHAMELEON）和动态策略（如 ODM、PiKE）。
静态方法在训练前确定权重，无法捕捉模型在长周期训练中的能力变化。
而早期的动态方法虽然能在线调整，但往往面临两难困境：要么计算开销巨大，拖慢训练速度；要么缺乏统一的框架，难以在不同架构和场景间灵活迁移。
AC-ODM 的核心动机就是打破这种权衡，提供一种既高效又灵活的动态混合方案。
### 核心 Insight：最大化梯度的“建设性干涉”
AC-ODM 没有使用启发式规则，而是引入了 Actor-Critic 架构。
- 环境（Environment）：大语言模型本身的训练过程。
- 状态（State）：包括迭代步数、各域 Loss、权重范数及其变化量等紧凑特征。
- 动作（Action）：输出各数据域的采样权重 α\alpha。
最精彩的设计在于 Reward 函数。
作者从优化几何角度证明，单纯最小化当前 Loss 并不足以加速全局收敛。他们定义了一个基于梯度对齐的奖励信号：衡量某个域的数据梯度与其余所有域梯度的内积。
直觉解释 ：如果某个域的数据更新方向与其他大部分数据一致，说明它在提供“建设性干涉”，能放大有效步长；反之则是“破坏性干涉”。AC-ODM 通过最大化这种对齐度，显式地优化了下降效率。
### 双模式设计：灵活应对不同场景为了兼顾工程落地，AC-ODM 提供了两种运行模式：
- 非代理模式（Non-Proxy）：端到端训练，Actor-Critic 与目标模型同步更新。适合从零开始预训练且无先验知识的场景，单步开销仅增加 0.4%。
- 代理模式（Proxy）：先在小型 Proxy 模型上学习策略，再迁移到大型目标模型。适合固定语料库的大规模训练，能避免大模型早期的探索噪声。
### 实验结果：速度与安全性的双重胜利在 Pythia-1B 和 LLaMA-0.9B 上的实验表现令人印象深刻。

反直觉发现 ：在 HumanEval 任务上的巨大提升，并非因为模型简单粗暴地增加了代码数据比例。日志显示，它实际上减少了 GitHub 数据的权重，转而提升了 StackExchange 等高质通用域的权重。这说明 AC-ODM 是通过优化全局梯度结构来增强推理能力，而非简单的数据堆砌。
### 工程启示与局限对工程师的建议：
- 细粒度分域是关键：实验表明，当数据域合并过粗时（如从 22 个合并为 5 个），AC-ODM 的效果会下降。因为正负梯度交互可能在组内抵消，导致奖励信号模糊。因此，保持数据域的语义差异性至关重要。
- 代理模式更经济：如果语料库固定且目标模型很大，使用小模型学习策略再迁移（Proxy Mode）是性价比最高的选择，整体加速可达 2.08x。
局限性：
该方法依赖于有意义的领域划分。如果数据本身同质化严重，或者缺乏清晰的域边界，AC-ODM 的优势将无法充分发挥。此外，它主要优化的是现有数据的混合比例，而非数据本身的清洗或去重，因此应与高质量的数据预处理流程配合使用。
总之，AC-ODM 证明了将强化学习引入数据工程不仅可行，而且高效。它为大模型预训练提供了一把更锋利的“手术刀”，让我们能用更少的算力，雕琢出更聪明的模型。
