目前的瓶颈不在模型架构，而在高质量训练数据的匮乏。
这篇来自 NVIDIA 的报告揭示了一个反直觉的事实：在桌面操作系统任务上，精心设计的合成数据竟然完胜最大规模的人类标注数据集。
## 为什么人类标注数据会“带偏”模型？
最大的公开资源 AgentNet 包含 2.25 万条人类标注轨迹，听起来很豪华。
但当我们用它在 UI-TARS 7B 上进行一轮监督微调（Supervised Fine-Tuning, SFT）时，OSWorld 基准测试的成功率从 26.3% 暴跌至 8–10%。
这就是典型的负迁移（Negative Transfer）。
原因很直接：人类标注数据大多局限于单应用、短路径（中位数仅 17 步），且缺乏复杂的多应用协同推理场景。
更糟糕的是，众包标注不可避免地引入了噪声。
模型学到的是“如何在简单任务上走捷径”，而不是“如何真正理解桌面状态”。
## ProCUA-SFT：用合成数据破局ProCUA-SFT 提出了一套全自动化流水线，生成了 310 万步、9.3 万条轨迹的合成数据。
其核心设计直觉在于解决合成数据的两大死穴： 任务不可行 和 上下文空洞 。
### 1. 前置条件验证：拒绝“幻觉”任务传统 LLM 生成任务时，经常让 Agent 打开一个不存在的文件，导致 rollout 浪费算力且产生错误示范。
ProCUA 引入了一位“裁判（Judge）”。
生成器在提出目标时，必须同时输出一组二元前置条件（如“桌面上是否存在 Q3.xlsx？”）。
只有当 Judge 独立验证所有条件为真时，任务才会进入执行阶段。
这种闭环机制确保了每个训练样本都是可执行的，从根源上杜绝了状态幻觉。
### 2. 真实内容注入：告别空壳桌面很多合成数据在空白文档或默认模板中生成，缺乏真实世界的复杂性。
ProCUA 注入了 912 个来自 SpreadsheetBench 的真实复杂电子表格（超 100 列、2 万行）以及约 1 万个开源演示文稿。
这使得 Agent 必须处理真实的公式、多表引用和非标准布局，从而学习到真正的跨应用推理能力。
### 3. 单模型闭环：消除规划-执行鸿沟生成目标和执行动作由同一个 VLM（Kimi-K2.5）完成。
这意味着模型只会生成自己能够完成的任务，最大化了轨迹的成功率，避免了“强规划、弱执行”带来的数据污染。
## 关键结果：数量与质量的双重胜利在 UI-TARS 7B 上微调一个 epoch 后，ProCUA-SFT 的效果令人印象深刻。
模型/数据集 OSWorld 成功率 对比基线提升 UI-TARS 7B (Base) 26.3% - + AgentNet SFT 8–10% -16.3 ~ -18.3 pp + ProCUA-SFT 45.0% +18.7 ppProCUA 不仅恢复了基线性能，还大幅超越了人类标注数据的效果。
此外，论文通过消融实验发现： 应用组合的多样性比动作类型的平衡更重要 。
按应用组合轮询采样（Round-robin by app combination）能取得最佳效果，这提示我们在构建多模态 Agent 数据时，应优先覆盖不同的软件交互场景。
## 工程启示与局限对工程师而言，这篇论文的价值在于验证了“数据质量 > 数据来源”的原则。
在 GUI Agent 领域，自动化合成数据不仅能突破人力标注的成本瓶颈，还能通过控制变量（如注入特定复杂文档）来定向增强模型能力。
不过，该方法目前主要依赖 Linux 桌面环境，且重度依赖强大的 VLM 作为生成器。
对于资源有限的团队，如何低成本复现这种“前置条件验证”机制，将是下一步落地的关键挑战。
