# ⭐⭐½ LLM 叙事能力取决于数据分布而非来源

**日期**: 2026-06-23

---

论文 : Characterizing Narrative Content in Web-scale LLM Pretraining Data链接 : https://arxiv.org/abs/2606.19468我们常以为给模型喂书或 Reddit 就能提升其讲故事的能力。但这篇论文揭示了一个反直觉事实：预训练数据的“叙事结构”是连续且多维的，简单的来源加权无法精准调控模型的叙事特质。
### 痛点：叙事数据黑盒大语言模型（LLM）的故事生成能力严重依赖预训练数据。然而，目前的数据策展（Data Curation）主要关注质量、去重和主题分布。我们几乎不知道数据中到底有多少“叙事内容”，以及这些内容是如何分布的。如果 Reddit 被降权，模型的叙事能力会受损吗？还是说它只是失去了某种特定类型的故事视角？现有研究对此缺乏系统性答案。
### 核心洞察：叙事是连续谱，而非二元标签论文最大的贡献在于将叙事从“有或无”的二元分类，转化为可测量的连续多维结构。作者基于叙事学理论，设计了包含 11 个可解释维度 的框架，涵盖三大核心要素：
- 能动性（Agency）：包括聚焦性（Focalization）、情绪、认知、状态改变和冲突。
- 场景（Setting）：包括具体性、时间锚定、空间锚定和感官细节。
- 事件关系（Events）：时间顺序和因果密度，以及每 token 的事件触发率。
⚠️ 关键直觉 ：不要假设“叙事来源”是均匀的。Reddit 虽然以故事闻名，但其内部叙事方差极大；而维基百科看似无叙事，却在特定维度上表现突出。
### 方法拆解：从理论到大规模标注为了在 3 万亿 token 的 DOLMA 语料库中量化这些特征，作者采用了一套高效的蒸馏流程：
- 分层采样：从 DOLMA 中提取约 1700 万个三句段落，利用二分类器筛选高叙事概率片段。
- 黄金标注：人工标注 400 个样本，验证维度间的一致性（平均 α=0.76\alpha = 0.760.76）。
- LLM 辅助标注：使用 Gemma-31B 对 5000 个样本进行大规模标注，作为教师模型。
- 模型蒸馏：训练轻量级 RoBERTa 分类器（NARRA BERT），将其性能对齐到人类标注水平，最终在约 300 万个段落上进行推断，生成 NARRA DOLMA 数据集。
### 关键发现：叙事能力的三维坐标通过主成分分析（PCA），作者发现叙事特征高度相关，并收敛为三个主要轴：
- PC1 内心独白（Interiority）：聚焦性、情绪和认知。Reddit 和古腾堡计划在此维度得分极高。
- PC2 具身事件（Grounded Eventfulness）：状态改变、冲突和时间/空间锚定。犯罪法律类和维基百科在此领先。
- PC3 世界纹理（Storyworld Texture）：具体性、感官细节。美食、时尚和旅游类文本占据主导。

⚠️ 反直觉结论 ：没有任何单一类别在所有三个轴上都占主导。Gutenberg 在内心独白和纹理上得分高，但在具身事件上表现中等。这意味着，单纯增加“书籍”数据的权重，并不能全面提升模型的所有叙事能力。
### 工程启示- 精细化数据混合：如果你希望模型擅长描写心理活动，增加 Reddit 或小说类数据是有效的；但如果需要模型处理具有强因果逻辑和时空锚定的事件序列（如新闻综述、法律案例），维基百科和犯罪类文本反而更重要。
- 避免粗粒度加权：当前的策展实践通常按来源（Source）或主题（Topic）进行粗粒度加权。论文显示，类别内部的叙事方差很大（例如 Reddit 的内部标准差高达 1.07）。仅靠来源标签无法捕捉叙事多样性。
- 评估指标升级：在评估模型故事生成能力时，应区分“内心独白丰富度”、“事件逻辑连贯性”和“场景描写细腻度”，而不是使用单一的“叙事质量”评分。
### 局限与展望NARRA DOLMA 是分层采样数据，故意过采样了叙事内容，因此绝对比例不能直接映射到原始 DOLMA 语料库。此外，事件关系分类器的 F1 分数（0.63）低于其 LLM 教师模型（0.78），表明事件结构的噪声较大。
这篇论文为理解预训练数据如何塑造 LLM 的叙事能力提供了首个细粒度地图。对于致力于提升模型创意写作或复杂推理能力的工程师来说，这是一份宝贵的数据审计指南。
