论文 : Sumi: Open Uniform Diffusion Language Model from Scratch链接 : https://arxiv.org/abs/2606.19005自回归（Autoregressive, AR）模型统治了 NLP 多年，但扩散语言模型（Diffusion LM）正试图打破这种单向生成的僵化。之前的扩散模型要么是掩码扩散（Masked Diffusion），一旦填好就不能改；要么是基于预训练 AR 模型微调的“伪”统一扩散。
东北大学团队发布的 Sumi 填补了这一空白：它是第一个从头预训练、参数规模达到 7B、且数据量高达 1.5T tokens 的 统一扩散语言模型（Uniform Diffusion Language Model, UDLM） 。
这意味着什么？意味着它允许在生成过程中的任何步骤修改任意 token，理论上具备更强的自我修正能力和更灵活的推理路径。对于工程师而言，Sumi 提供了一个干净的基准，让我们能真正研究原生统一扩散的缩放律和生成动力学。
### 为什么需要 Sumi？
现有的扩散语言模型存在两个主要痛点：
- 掩码扩散的不可逆性：如 LLaDA 等模型，一旦某个位置被填充，后续步骤无法再修改。这限制了模型的“后悔”能力。
- 缺乏原生大规模训练：之前的 UDLM 要么规模太小（<2B），要么是基于 AR 模型适配而来（如 DiffusionGemma）。我们不知道在数据富集 regime 下，原生的统一扩散表现如何。
Sumi 的目标很明确：提供一个完全开源、从头训练的 7B UDLM，作为研究统一扩散缩放行为和控制能力的参考点。
### 核心设计拆解Sumi 并没有发明全新的架构，而是对现有最佳实践进行了严谨的工程化落地。
- 架构选择：采用标准的 LLaMA-style 双向 Transformer（36 层，隐藏维度 4096）。关键在于它使用了 GIDD（Generalized Interpolating Discrete Diffusion） 框架，并采用信噪比（SNR）重参数化形式。
- 训练策略：
预训练：1.3T tokens，序列长度 1,184。数据主要来自 llm-jp-corpus-v4 的英文部分，经过教育分数过滤。
- 中期训练（Mid-training）：250B tokens，重点增强代码、数学和推理能力。这一步将序列长度扩展到 4,864，并使用了 WSD 学习率调度。
- 去偏见技巧：为了防止注意力汇聚在特定位置（Attention Sink），Sumi 在注意力机制中引入了“off-by-one softmax”。这是一个细微但重要的工程细节，有助于提升训练稳定性。
### 关键结果：强于知识，弱于常识Sumi 在多个基准测试中与同量级的 AR 模型（如 Falcon-7B, Llama 2-7B）进行了对比。以下是核心数据对比（基于论文 Table 1）：

### 反直觉的发现：生成动力学探针论文最有趣的部分在于对生成过程的深入探测（Section 4），揭示了 UDLM 的一些独特行为：
-画布长度敏感性：Sumi 的生成流畅度高度依赖于“画布长度”（Canvas Length，即初始分配的 token 缓冲区大小）。
在训练范围内（约 1024-4096），模型表现稳定。
- 一旦超出此范围，尤其是过短的画布，GSM8K 等数学任务的困惑度会爆炸式增长。这表明 UDLM 对上下文窗口的依赖比 AR 模型更敏感。
-置信度采样诱导了顺序：
UDLM 理论上是“无序”的（任何 token 都可更新）。但实验发现，使用**置信度自适应采样（Confidence-based Adaptive Sampling）**时，模型会自发形成一个“承诺顺序”。
- 如图 3 所示，高置信度的位置先被确定，低置信度的后确定。这种自组织行为使得 UDLM 在代码生成任务上可以实现有限的并行解码（每步更新 4 个 token 而不损失精度），但在数学推理上依然需要串行。
-自我修正的幻觉：
理论上，UDLM 可以无限次修改已生成的 token。但实验表明，增加额外的去噪步骤（Revision Budget）并不能带来自我修正。
- 大多数“修改”只是 A→B→AA \rightarrow B \rightarrow AB→A 的无意义往返，最终答案几乎从未翻转。这说明当前的 UDLM 并不具备真正的“反思”能力，额外计算纯属��费。
### 工程启示对于想要尝试非自回归模型的工程师，Sumi 提供了几个关键指导：
- 数据配比决定短板：如果你追求极致的知识和代码能力，可以模仿 Sumi 的教育/代码重加权策略。但必须接受常识能力的下降。
- 采样器至关重要：不要使用简单的祖先采样（Ancestral Sampling）。在 UDLM 中，置信度自适应采样不仅能提升质量，还能引入隐含的结构化顺序，从而允许一定的并行加速。
- 固定画布长度：在实际部署中，建议将 Canvas Length 固定在训练分布的中心值（如 2048），避免动态调整导致性能崩塌。
### 局限与展望Sumi 目前仍是一个基础模型（Base Model），未经过指令微调（SFT）。这意味着它直接用于对话可能效果不佳。此外，其常识能力的缺失表明，单纯增加教育类数据并非万能钥匙，未来的 UDLM 需要在数据多样性上做得更好。
总的来说，Sumi 证明了统一扩散在大规模下是可行的，且在特定领域能超越传统 AR 模型。但它也揭示了 UDLM 目前仍缺乏真正的自我修正能力。对于研究者来说，这是一个宝贵的开源基准；对于工程师来说，它提示我们：在非自回归生成中，采样策略比架构本身更值得优化。
