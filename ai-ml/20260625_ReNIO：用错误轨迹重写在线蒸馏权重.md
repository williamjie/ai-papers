论文 : ReNIO: Reweighting Negative Trajectory Importance for LLM On-Policy Distillation链接 : https://arxiv.org/abs/2606.23104在线策略蒸馏（On-Policy Distillation, OPD）正成为推理模型后训练的主流范式，因为它比强化学习更省算力、反馈更密集。但现有方法有个致命盲点：它把学生生成的“正确”和“错误”轨迹一视同仁。这篇论文反直觉地发现， 错误轨迹其实更有价值 ，并提出了一种无需完整答案即可识别高价值错误的方法 ReNIO。
### 痛点：为什么标准 OPD 效率低下？
OPD 的核心逻辑是让学生模型（Student）生成输出，然后让教师模型（Teacher）提供逐 Token 的监督信号。相比强化学习（RL），OPD 不需要等待最终答案就能训练，因此可以使用截断的前缀（Prefix），大幅降低显存和时间成本。
然而，标准 OPD 对所有学生生成的轨迹（SGO）赋予相同的训练权重。这很浪费。
⚠️ 反直觉发现 ：论文通过控制变量实验发现，在 OPD 和在线自蒸馏（OPSD）中， 仅使用错误轨迹训练的效果，始终优于仅使用正确轨迹训练 。
- 在 AIME24 数据集上，仅用错误轨迹训练的 OPD 比仅用正确轨迹高出 +3.60%。
- 行为分析显示，仅用正确轨迹训练的模型倾向于生成更短的回答，且缺乏“反思”标记（如 “wait”, “check”）；而错误轨迹能保留更多探索性和自我修正的推理过程。
这意味着，学生模型在能力边界附近的“失败尝试”，包含了比“成功路径”更丰富的纠错信号。问题在于：如何在不生成完整答案（从而破坏 OPD 的低成本优势）的前提下，识别出这些高价值的错误轨迹？
### 方法拆解：ReNIO 的核心直觉ReNIO 的设计非常优雅，它不依赖最终答案的对错，而是利用 学生与教师在前缀阶段的概率分歧 来加权。
-定位关键分歧点（Pivotal Tokens）：
推理轨迹往往因为少数几个关键 Token 的选择错误而走向歧途。ReNIO 计算每个 Token 的学生-教师对数概率比（Log Ratio）：
ℓt=log⁡πS(yt∣x,y<t)πT(yt∣x,y<t)\ell_t = \log \frac{\pi_S(y_t | x, y_{<t})}{\pi_T(y_t | x, y_{<t})}​=logπT​(yt​∣x,y<t​)πS​(yt​∣x,y<t​)​如果学生给某个 Token 的概率很高，而教师认为它很低（ℓt\ell_t​ 很大），说明学生在这里做出了一个“自信但被教师拒绝”的分支决策。这就是潜在的错误根源。
-阈值筛选与聚合：
长尾分布处理：实验显示，大多数 Token 的对数比接近 0（师生一致），只有少数关键 Token 比值极大。ReNIO 设定一个固定阈值 τ\tau，只保留超过阈值的“关键 Token”。
- 几何平均聚合：将选中 Token 的对数比进行几何平均，得到该轨迹的原始权重 w(x,y)w(x, y)。这确保了只有包含显著分歧信号的轨迹才会获得高权重。
-批次归一化（Batch Normalization）：
为了防止梯度尺度失控，ReNIO 在 Batch 内对权重进行归一化，使平均权重为 1。这样既重新分配了训练重点，又保持了整体更新规模的稳定。
### 关键结果：低成本下的显著增益ReNIO 在数学推理和代码生成任务上均表现出一致的提升，且完全保留了 OPD 使用短前缀的优势。

- 训练时间：使用 1024 Token 前缀的 OPD/OPSD 比 GRPO（需完整 rollout）快得多。即使将前缀延长到 4096 Token，ReNIO 仍能带来额外提升（例如 Qwen3-1.7B OPSD 从 40.83 提升至 42.78）。
- 消融实验：移除阈值筛选或批次归一化都会导致性能下降。特别是批次归一化，缺失时平均得分从 42.04 降至 39.54，证明其对于稳定训练至关重要。
### 工程启示- 拥抱错误：在微调推理模型时，不要盲目过滤掉低质量或错误的生成样本。这些“失败案例”往往蕴含着模型当前能力边界的宝贵信息。
- 低成本增强：ReNIO 无需额外的奖励模型（Reward Model）或完整答案验证，仅利用前缀概率即可计算权重。这意味着你可以在现有的 OPD/OPSD 流水线中轻松集成此模块，几乎不增加额外开销。
- 稳定性提升：对于小模型（如 1.5B-1.7B），标准 OPSD 有时会导致性能波动甚至下降。ReNIO 通过强调高信息量的负样本，显著稳定了训练过程，使小模型也能从自蒸馏中受益。
