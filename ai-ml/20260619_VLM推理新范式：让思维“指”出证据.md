# ⭐⭐ VLM推理新范式：让思维“指”出证据

**日期**: 2026-06-19

---

论文 : Thinking with Visual Grounding链接 : https://arxiv.org/abs/2606.16122纯文本的 Chain-of-Thought (CoT) 在视觉任务中正在失效。模型虽然能写出逻辑通顺的推理过程，但往往无法自证其词，导致“幻觉式推理”泛滥。这篇论文提出了一种工程上极具落地价值的解法：强制模型在推理时输出坐标标签（Bounding Box 或 Point），将思维与图像像素强行绑定。
### 痛点：不可验证的视觉思考现有的 VLM 推理大多停留在自然语言层面。当模型说“左边的红色汽车”时，它并没有真正“看”到那辆车，只是基于概率生成了这句话。这种纯文本推理存在两个致命缺陷：
- 难以监督：最终答案正确，不代表推理路径正确。模型可能蒙对答案，但中间过程全是胡扯。
- 缺乏空间约束：语言无法精确表达复杂的几何关系（如遮挡、相对距离），导致空间推理能力上限很低。
### 核心 Insight：思维必须“落地”
论文的核心直觉非常直观： Visual thinking should not only sound right; it should show its evidence.
作者提出了 Visually Grounded Thinking，要求模型在生成推理文本时，穿插输出带有坐标的 grounding tag。
- Box Mode: <obj> black laptop | [x1, y1, x2, y2] </obj>- Point Mode: <obj> black laptop | [x, y] </obj>这不仅仅是格式改变，更是训练范式的革新。作者构建了一个基于 SAM3 的自动化数据合成流水线：先用强模型生成正确推理，再用 Agent 提取关键物体并生成 RLE Mask，最后转化为 Box/Point 监督信号。这种“蒸馏+Agent”的数据构造方式，解决了高质量 Grounding 数据稀缺的工程难题。
### 强化学习中的精细奖励设计最精彩的部分在于 RL 阶段的 Reward Design。传统的 RL 只给最终答案打分（Sparse Reward），而这里引入了 Grounding Reward ：
- Box IoU: 计算生成框与真值框的交集并比，提供平滑的梯度信号。
- Point F1: 检查生成的点是否落在目标 Mask 内部，解决实例级定位问题。
作者还设计了一个轻量级 VLM Router，用于匹配模型生成的物体名称与 Ground Truth。这种解耦设计避免了因命名差异导致的奖励误判，是工程落地时的关键细节。
### 实验结果：小模型逆袭大模型在 Gemma3-4B-IT 基座上的实验结果令人印象深刻。引入 Grounded Thinking 后，4B 模型在空间推理任务上直接追平甚至超越了同系列的 27B 模型 。
Benchmark Gemma3-4B (Base) Grounded Box (+RL) Gemma3-27B (Ref) VSR-zeroshot 56.65% 68.08% 69.25% EmbSpatial 57.94% 79.13% 87.64% SpatialMQA 57.62% 67.66% 65.14%⚠️ 反直觉发现 ：在计数任务（Counting）中，Point Grounding 表现优于 Box；而在空间关系推理中，Box Grounding 收益更大。这是因为计数只需要实例定位，而空间关系依赖物体的几何边界信息。
### 工程启示与局限- RL 稳定性提升：纯文本 CoT 在 RL 训练中容易出现“长度坍塌”（Length Collapse），导致探索能力下降。Grounded Thinking 通过强制插入坐标标签，为生成过程提供了额外的局部结构约束，显著稳定了训练曲线。
- 数据合成可复用：论文开源的 SAM3-based Agent 流水线具有极高的复用价值。任何需要视觉定位微调的场景，都可以参考这种“推理蒸馏+Agent 标注”的模式。
- 局限：Point Reward 的信号较为离散（Piecewise constant），优化难度高于 Box IoU。此外，目前主要验证于计数和空间推理，在更复杂的因果推理中效果尚待观察。
这篇论文证明了： 让模型“指”出证据，比让它“说”出理由更重要。 对于正在构建视觉 Agent 或微调 VLM 的工程师来说，Grounded Thinking 是一个值得立即尝试的低成本、高回报方案。
