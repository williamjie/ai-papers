# 告别 KV 爆炸：Unlimited OCR 实现超长文档一次性解析

**日期**: 2026-06-23

---

论文 : Unlimited OCR Works链接 : https://arxiv.org/abs/2606.23050当前端到端（End-to-End）OCR 模型面临一个尴尬的工程死结：随着输出序列变长，KV Cache 线性膨胀，导致显存爆满且推理速度断崖式下跌。百度提出的 Unlimited OCR 通过一种极简的注意力机制改造，实现了在标准 32K 上下文窗口下，一次性无损解析数十页文档。
### 痛点：LLM 做 OCR 的“记忆负担”
传统 OCR 采用检测加识别的分步流水线（Pipeline），而基于大语言模型（LLM）的端到端方案虽然精度更高，却引入了严重的效率问题。
在标准自回归解码中，每生成一个 Token，模型都需要访问之前所有的 Key-Value 对。对于 OCR 这种长文本转录任务，输出长度往往是输入视觉 Token 的数十倍。
核心矛盾 ：人类抄写书籍时，只需关注原文和刚写的几个字；但现有 LLM 必须记住所有已生成的文字，导致计算开销随长度线性增长，根本无法处理长文档。
### 方法拆解：参考滑动窗口注意力（R-SWA）
Unlimited OCR 的核心创新在于用 Reference Sliding Window Attention (R-SWA) 替换了解码器中所有的标准多头注意力（MHA）。
其设计直觉非常反直觉且优雅：
- 全局可见：每个生成的 Token 都可以看到所有的“参考 Token”（即图像视觉特征和 Prompt）。这部分信息是静态的，不需要随着生成过程更新状态。
- 局部滑动：对于已经生成的文本 Token，模型只关注最近 nn 个（默认 128）Token。
这种设计模拟了人类的工作记忆：眼睛盯着书（全局参考），手边只看刚写的几个字（局部滑动）。
工程价值在于 KV Cache 的恒定化 ：
- 传统 MHA：KV Cache 大小 C=Lm+TC = L_{m} + TLm​+T，随生成长度 TT 无限增长。
- R-SWA：KV Cache 大小 C=Lm+min⁡(n,T)C = L_{m} + \min(n, T)Lm​+min(n,T)，上限被严格锁定为 Lm+nL_{m} + n​+n。
这意味着无论解析多少页文档，显存占用和计算复杂度始终保持恒定，彻底消除了长序列推理中的“越往后越慢”现象。
### 关键结果：精度与速度的双重提升基于 DeepSeek OCR 基线，Unlimited OCR 在 OmniDocBench v1.5 上取得了显著优势。以下是核心数据对比：
指标 DeepSeek OCR (基线) Unlimited OCR 变化幅度 Overall Score 87.01% 93.23% +6.22% Text Edit Dist. 0.073 0.038 -47.9% (越低越好) Table TEDS 84.97% 90.93% +5.96%在推理效率方面，优势更为明显。随着输出长度增加，基线模型的吞吐量（TPS）持续下降，而 Unlimited OCR 保持稳定：
- 256 Token 时：两者速度几乎持平（~7229 TPS）。
- 6144 Token 时：基线模型降至 5822 TPS，而 Unlimited OCR 保持在 7847 TPS。
- 结论：在长序列场景下，Unlimited OCR 比基线快约 35%。
更令人印象深刻的是长文档解析能力。在输入超过 40 页的测试中，模型依然保持了低于 0.11 的编辑距离和 97% 的 Distinct-35 多样性指标，证明了 R-SWA 在处理超长依赖时的有效性。
### 工程启示对于从事文档智能（Document AI）或长文本处理的工程师，这篇论文提供了两个重要信号：
- 注意力机制并非不可妥协：在特定任务（如 OCR、ASR、翻译）中，输出 Token 之间的长距离依赖其实很弱。强制使用全注意力不仅浪费算力，还可能引入噪声。
- 架构设计的通用性：R-SWA 不仅仅适用于 OCR。任何具有“参考输入”和“长序列输出”特征的任务（如语音识别 ASR、机器翻译），都可以尝试用这种恒定 KV Cache 的设计来突破长度限制。
### 局限与展望目前 Unlimited OCR 仍受限于预填充（Prefill）阶段的视觉 Token 数量。虽然 DeepEncoder 实现了 16 倍的压缩率，但在处理极长文档时，前缀长度依然可观。
作者计划未来通过构建“预填充池”，让模型学习自动获取所需的 KV 块，从而模拟人类翻页的行为，实现真正的无限长 OCR。在此之前，对于本地部署场景，Unlimited OCR 已经是一个极具吸引力的 SOTA 选择。
