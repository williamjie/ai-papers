**日期**: 2026-06-15

---

论文 : VISTA: View-Consistent Self-Verified Training for GUI Grounding链接 : https://arxiv.org/abs/2606.14579做 GUI Agent 的工程师都知道，强化学习（Reinforcement Learning, RL）是提升定位精度的关键。但直接套用 GRPO 往往效果不佳，因为截图场景太特殊了。这篇来自蚂蚁集团和浙江大学的 VISTA 论文，提供了一个非常巧妙的工程解法：通过“视图一致性”重构对比组，解决了 GRPO 在 GUI 任务中的奖励坍缩问题。
### 为什么标准 GRPO 在 GUI 上会失效？
GRPO 的核心逻辑是通过一组采样（Rollouts）的相对优势来更新策略。但在 GUI Grounding 任务中，这个机制经常“死机”。
痛点 ：固定视图下的同质化奖励。
对简单样本，模型所有采样都点中了，奖励全是 1；对困难样本，全都没点上，奖励全是 0。
这就导致组内方差 σ=0\sigma = 0 0 ，相对优势（Advantage）全部归零。论文数据显示，在固定视图训练下， 不到 5% 的样本能形成包含成功和失败混合的有效对比组。没有方差，就没有学习信号，模型自然无法进步。
### VISTA 的核心直觉：同题不同卷VISTA 的设计直觉非常清晰：既然同一张图看不出差别，那就换几张图来考同一个知识点。
1. 视图一致性分组（View-Consistent Group Rollout）
VISTA 不再对单张截图重复采样，而是对同一个 GUI 实例生成多个“目标保留”的裁剪视图（Target-preserving views）。
- 怎么做：通过裁剪保持目标元素可见，并精确重映射坐标框。
- 为什么有效：语义不变（还是点那个按钮），但几何位置变了。模型在不同视图中可能成功也可能失败，从而在组内制造出必要的奖励方差。
- 效果：这一改动直接将有效对比组的比例从 <5% 提升到了 20% 左右，恢复了 RL 的学习动力。
2. 自验证跨视图锚点（Self-Verified Cross-View Anchoring）
多视图采样虽然解决了方差问题，但也带来了新麻烦：坐标格式容易乱，训练不稳定。
VISTA 引入了一个“保底”机制，但设计得非常克制：
- 条件激活：只有当组内至少有一个模型采样成功（获得最大奖励）时，才引入 Ground Truth 作为锚点。
- 排除基线：这个锚点不参与组统计（μ,σ\mu, \sigma），仅作为优势加权监督信号。
- 核心逻辑：如果模型在所有视图都失败了，说明当前策略还没摸到门道，强行喂标准答案会变成无条件的监督微调（SFT），破坏 RL 探索；只有当模型“开窍”时，才用标准答案稳固成果。
### 实验数据：实打实的提升在 ScreenSpot-Pro 这个对坐标精度要求极高的基准上，VISTA 的效果非常显著：
模型规模 基线 (Qwen3-VL) VISTA 提升后 绝对增益 4B 55.5% 63.4% +7.9 8B 52.7% 65.8% +13.1 30B-A3B 53.7% 67.0% +13.3在更难的 OSWorld-G-R 和 OSWorld-G 基准上，平均得分也分别从 71.1/69.0/73.6 提升到了 75.5/76.3/77.6 。
反直觉发现 ：
单纯的裁剪增强（SFT + aug）对 ScreenSpot-Pro 的提升微乎其微（仅 +0.7%）。这说明 VISTA 的核心价值不在于数据增强本身，而在于 利用多视图差异重构 RL 的对比信号 。
### 工程启示与局限对于正在微调 GUI Agent 的团队，VISTA 提供了两个关键建议：
- 重视奖励方差：不要盲目堆采样次数，如果所有采样结果一致，增加数量毫无意义。通过数据变换（如裁剪、缩放）引入几何多样性，是激活 GRPO 的关键。
- 谨慎使用 Oracle：在 RL 阶段直接混入标准答案往往有害。VISTA 的“自验证”门控机制值得借鉴——只在模型表现出一定能力时进行监督，避免过早坍塌到模仿学习。
局限性 ：
VISTA 目前主要针对可操作的 GUI Grounding 任务。对于包含拒绝回答（Refusal）或复杂推理的混合数据集，需要额外的路由机制来区分任务类型，否则锚点机制可能会引入噪声。此外，多视图裁剪增加了约 25% 的训练耗时，需要在精度和成本间做权衡。
