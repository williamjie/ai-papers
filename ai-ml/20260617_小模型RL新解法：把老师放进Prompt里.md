# ⭐ 小模型RL新解法：把老师放进Prompt里

**日期**: 2026-06-17

---

论文 : Zone of Proximal Policy Optimization: Teacher in Prompts, Not Gradients链接 : https://arxiv.org/abs/2606.18216在端侧部署大模型时，我们常面临一个尴尬：小模型（如 0.8B-2B）通过强化学习（RL）训练时，遇到难题往往“全军覆没”，导致梯度为零、无法学习。这篇来自 NVIDIA 的论文提出了一种极具巧思的解法： ZPPO（Zone of Proximal Policy Optimization） 。它不直接让模型模仿老师的答案，而是把老师的正确回答和学生的错误回答一起塞进 Prompt，让学生自己去“挑错”。
这种方法不仅解决了小模型 RL 训练中的“零优势”死锁，还在保持 On-Policy 假设的前提下，显著提升了泛化能力。
### 痛点：蒸馏太脆，RL 易飘现有的知识迁移方案主要有两条路，但在小模型场景下都有硬伤：
- 知识蒸馏（Distillation）：让小模型模仿大模型的 Logits。这本质上是监督拟合，小模型容易死记硬背老师的“尖锐模式”，导致在训练集之外的基准测试上泛化能力极差。
- 强化学习（RL）：利用学生自己的 Rollout 进行训练。但问题是，对于小模型来说，很多难题它的每次尝试都是错的（Zero Success）。这意味着所有 Rollout 的 Advantage 都是 0，梯度直接消失。如果强行把老师的正确回答注入梯度，又会破坏 On-Policy 假设，导致策略漂移（Policy Drift）。
ZPPO 的核心 Insight 来自维果茨基的“最近发展区”理论： 老师不应该出现在梯度里，而应该出现在 Prompt 里。
### 方法拆解：BCQ 与 NCQ 的双剑合璧ZPPO 在 GRPO 的基础上，针对那些学生准确率低于 50% 的“难题”，构建了两种特殊的 Prompt 结构：
- 二元候选问题（BCQ）：
做法：抽取一个老师的正确回答和一个学生的错误回答，匿名化后作为两个选项放入 Prompt。
- 直觉：让学生扮演“裁判”，在两个选项中推理并选择正确的一个。因为学生最终生成的答案是自己采样的，所以梯度依然是 On-Policy 的，避免了漂移。
- 负向候选问题（NCQ）：
做法：把学生在同一问题上的所有错误回答聚合在一起，明确告诉模型“这些全错了”，并要求给出正确答案。
- 直觉：在标准 RL 中，独立的错误 Rollout 互不可见。NCQ 让模型看到自己失败的共性模式，从而针对性地修正逻辑漏洞。
此外，ZPPO 引入了一个 Prompt Replay Buffer 。难题会被反复重放，直到学生在该题上的平均准确率超过 50%（“毕业”），或者被 FIFO 淘汰。这种机制确保了训练算力始终集中在学生“跳一跳够得着”的最近发展区内。
### 关键结果：小模型增益显著论文在 Qwen3.5 系列上进行了广泛测试，对比了 Off-Policy/On-Policy Distillation 和 GRPO。数据非常有说服力：
模型规模 16个 VLM 基准平均提升 10个 LLM + 5个 Video 基准平均提升 Qwen3.5-0.8B +9.3 pp (vs Base) +6.8 pp (vs Base) Qwen3.5-2B +5.2 pp (vs Base) +4.3 pp (vs Base) Qwen3.5-9B +2.8 pp (vs Base) +2.7 pp (vs Base)
⚠️ 反直觉发现 ：传统的蒸馏方法在 VLM 训练集上可能略有提升，但在 LLM 和 Video 等跨模态泛化任务上，反而会 降低 性能（0.8B 模型下降 -2.5 pp）。而 ZPPO 在所有基准上均实现了正向增益。
消融实验显示，BCQ、NCQ 和 Replay Buffer 三者缺一不可。单独使用 Replay Buffer 仅带来 +0.5~1.6 pp 的微幅提升，只有与 BCQ/NCQ 结合时，才产生超加性的效果（Super-additive）。
### 工程启示：微调小模型的三个细节除了核心算法，论文还分享了一些对工程落地极具价值的 RL Recipe 细节：
- 迭代次数（Iterations per step）：GRPO 默认每步迭代 16 次，但这会导致严重的策略漂移。实验表明，4 次迭代是平衡漂移与训练效率的最佳甜点（Sweet Spot）。
- 优势归一化（Advantage Normalization）：在计算 Batch 级别的 Advantage 标准差时，必须排除那些全对或全错（零优势）的 Group。如果不排除，零值会压缩标准差，导致其他样本的优势被异常放大，破坏训练稳定性。
- 移除 KL Penalty：在小模型 RL 中，移除参考策略的 KL 惩罚项有助于释放模型的探索能力。
### 局限与展望ZPPO 并非万能。它的上限受限于老师的能力：如果连老师都答错的题，BCQ 就无法生成候选项，只能依赖 NCQ。这意味着当学生能力接近老师时，BCQ 的贡献会边际递减。未来的方向可能是引入多老师集成或合成数据，以扩展这个“最近发展区”。
对于致力于端侧智能、需要高效训练小模型的团队来说，ZPPO 提供了一种无需复杂梯度操作、仅通过 Prompt 工程即可显著提升 RL 效果的优雅方案。
