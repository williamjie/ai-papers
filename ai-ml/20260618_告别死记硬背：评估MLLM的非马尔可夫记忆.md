# ⭐⭐⭐ 告别死记硬背：评估 MLLM 的非马尔可夫记忆

**日期**: 2026-06-18

---

论文 : Beyond the Current Observation: Evaluating Multimodal Large Language Models in Controllable Non-Markov Games链接 : https://arxiv.org/abs/2606.19338当多模态大语言模型（MLLM）被部署为闭环策略时，它们面临的真实挑战往往不是“看懂当前画面”，而是“记住刚才发生了什么”。
现有的基准测试要么像围棋那样信息全透明，要么像长文本问答那样只考事后回忆。这篇论文提出的 RNG-Bench 直击痛点：它构建了一个“非马尔可夫”环境，强制模型必须基于历史重建隐藏状态才能做出正确决策。对于正在开发具身智能 Agent 的工程师来说，这是一个极具参考价值的诊断工具。
### 为什么现有基准不够用？
在传统的马尔可夫游戏中（如国际象棋），当前棋盘状态足以决定下一步行动。但在真实交互中，关键信息往往一闪而过。
论文区分了两种场景：
- Remember-to-Answer：看完整个轨迹后回答问题。错误没有后果，只是分数低一点。
- Remember-to-Act（本文核心）：每一步的错误决策都会改变后续观察，导致误差累积甚至死循环。
现有的 Agent 基准通常将记忆、探索、规则理解混在一起，无法定位失败原因。RNG-Bench 通过两个极简游戏剥离了这些干扰因素。
### 方法拆解：可控的非马尔可夫博弈RNG-Bench 包含两个互补环境，分别测试不同类型的隐状态跟踪能力：
- Matching Pairs（配对卡片）：测试静态离散记忆。卡牌身份仅在翻开时可见，随后翻回背面。模型必须记住“哪个位置是什么牌”，并在后续步骤中调用这些记忆。
- 3D Maze（3D 迷宫）：测试动态空间记忆。模型只有第一人称视角，必须通过局部视图增量更新全局地图、自身位置和朝向。
核心设计直觉：
论文引入了一个名为 Memory Gap 的指标。它对比了模型在正常条件下的得分与注入“上帝视角”（Oracle）隐藏状态后的得分。
- 如果 Memory Gap 很大，说明瓶颈在于遗忘（记忆重建失败）。
- 如果 Memory Gap 很小，说明瓶颈在于决策（即使知道真相也选错动作）。
这种设计让工程师能清晰判断：模型是“脑子不好使”还是“记性太差”。
### 关键结果：前沿模型依然脆弱在最高难度配置下（约 128K tokens 上下文和 350 张图像输入），即便是 GPT-5.4 和 Gemini-3.1-Pro 这样的顶级模型也远未饱和。

视觉感知是主要瓶颈 。当输入从纯文本变为复杂图像时，性能断崖式下跌。这说明模型并非无法处理长上下文，而是难以从噪声中提取并绑定关键特征到历史中。
### 工程启示：微调与外部记忆论文证明了监督微调（Supervised Fine-Tuning, SFT）的有效性。使用最优策略生成的轨迹对 Qwen3.5-9B 进行微调后：
- RNG-Bench 性能提升：Matching Pairs Score% 从 0% 提升至 29.5%。
- 正迁移效应：在外部记忆基准 EMeMBench 上提升了 +5.2%，且未损害通用多模odal 能力（平均仅变化 +0.5%）。
此外，实验显示提供外部记忆（如小地图或已知卡片列表）能显著缩小 Memory Gap。这提示我们在构建 Agent 时， 显式的状态管理模块可能比单纯依赖模型内部注意力机制更可靠 。
### 局限与展望RNG-Bench 目前仅覆盖两种环境，且视觉风格较为单一。Memory Gap 指标依赖于 Oracle 注入，这是一种近似诊断而非因果分解。但对于希望提升 Agent 长期交互稳定性的团队而言，这是一个低成本、高信度的测试起点。
