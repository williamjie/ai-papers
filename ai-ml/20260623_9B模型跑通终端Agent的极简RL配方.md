论文 : Tmax: A simple recipe for terminal agents链接 : https://arxiv.org/abs/2606.23321终端 Agent（Terminal Agents）正在成为大模型落地的下一个“深水区”，但学术界一直缺乏简单、可复现且有效的强化学习（Reinforcement Learning, RL）基线。这篇来自 Allen AI 和华盛顿大学的 Tmax 论文，用一套极简的配方证明了：你不需要万亿参数，9B 模型配合精心设计的合成数据，就能在终端任务上吊打大多数开源竞品，甚至逼近闭源小模型。
### 痛点：现有方案太“虚”
之前的终端 Agent 研究要么停留在简单的 Bug 修复（如 SWE-Bench），要么生成的训练数据过于简单（如 Endless Terminals 中 Gemini-3-Flash 的通过率高达 90%）。更致命的是，缺乏针对长程、复杂终端任务的 RL 训练基线。大多数工作只做监督微调（Supervised Fine-Tuning, SFT），或者使用了极其复杂的验证流程，导致数据难以规模化，且模型容易过拟合特定的评测 Harness。
### 核心 Insight：组合式采样与“软过滤”
Tmax 的核心贡献在于其数据生成管线和 RL 训练细节的极简主义设计。
-分层组合采样（Hierarchical Sampling）：
他们没有依赖昂贵的教师模型验证，而是通过 9 个正交轴（领域、技能、复杂度、Persona 等）进行组合采样。这种设计带来了两个好处：
多样性：通过 Persona（如“红队操作员”）和多媒体 Fixture（图片、音频、二进制文件），强制 Agent 使用 OCR、ffmpeg 等工具，模拟真实世界的复杂输入。
- 难度控制：显式地控制任务复杂度（从几条命令到 30-60 条命令的工作流），避免了以往数据“要么太简单要么完全不可解”的双峰分布。
-软过滤机制（Soft Filtering）：
这是一个非常工程化的直觉。传统做法是用强模型验证每个生成任务的质量，成本极高。Tmax 直接跳过这一步，依赖 RL 训练过程中的自然筛选——如果某个任务策略通过率始终为 0，它就不会产生梯度贡献。实验显示，这种“懒惰”的过滤方式效率极高，每批次仅过滤不到 8 个样本。
### 关键结果：小模型的大胜利Tmax-9B 在 Terminal-Bench 2.0 上达到了 27% 的通过率。这个数字看似不高，但在同等规模下极具统治力：
- 超越竞品：在 <32B 参数区间，Tmax-9B 的性能优于 LiberCoder-32B、TerminalTraj-14B 等模型，并接近 Claude-Haiku-4.5。
- 数据对比：在 Terminal-Bench Lite 上，使用 Tmax-15K 数据训练的模型得分为 57.2%，显著高于使用 Endless Terminals（52.6%）和 OpenThinker-Agent（53.0%）数据的模型。

### 工程启示：稳定性是 RL 的生死线这篇论文最宝贵的部分在于其对 RL 训练不稳定性的坦诚分享。终端任务通常涉及 20+ 步的多轮交互，极易导致训练崩溃。作者给出了三个关键调优建议：
- FP32 LM Head：在推理和训练期间保持语言模型头部的 FP32 精度，以消除数值不匹配导致的 LogProb 尖峰。这在 Qwen 3.5 系列模型中尤为关键。
- DPPO 算法：相比 naive GRPO，使用 Divergence Proximal Policy Optimization (DPPO) 能有效限制策略更新幅度过大，防止训练坍塌。
- 大 Group Size：将 Group Size 设置为 32（而非常见的 8），能显著平滑奖励信号，提升训练稳定性。
### 局限与展望Tmax 目前主要依赖 Gemini-3-Pro 进行数据生成，这引入了对前沿模型的依赖。此外，虽然其在 Terminal-Bench 上表现优异，但在更通用的代码基准（如 SWE-Bench Verified）上仅提升了约 5 个点，说明终端特定技能的泛化仍有边界。
对于工程师而言，Tmax 提供了一套可复现的“最小可行产品”：如果你想在本地部署一个能写脚本、跑服务的 Agent，与其盲目堆砌数据量，不如关注数据的难度分布控制和 RL 训练的数值稳定性。
## 📝 AI 点评点评时间：2026-06-23 18:15 ｜ reviewer: DeepSeek V4 Flash核心贡献: 论文提出了 TMAX，一个简单可复现的终端 agent RL 训练配方，其核心包括通过分层组合采样生成的 14,600 个 RL 环境数据集 TMAX-15K，以及基于 DPPO、FP32 LM head 等稳定性技巧的 RL 训练流程，最终 TMAX-9B 在 Terminal-Bench 2.0 上达到 27% 的通过率，在 <32B 参数开源模型中取得最优。
亮点:
- 博文准确提炼了数据生成管线中的“分层组合采样”与“软过滤”两大工程直觉，并指出其跳过昂贵教师验证、依赖 RL 自然筛选的低成本特性，与原文 §3.1 中的描述一致。
- 博文抓住了 RL 训练稳定性这一关键工程价值点，明确列出 FP32 LM head、DPPO、大 Group Size 三个具体调优建议，并解释了各自作用（如消除 LogProb 尖峰、限制策略更新幅度、平滑奖励信号），这些在原文 §5.2 和 §4.1 中有详细实验支撑。
- 博文注意到“反直觉发现”——对强基座模型（Qwen 3.5）直接进行 RL 比先 SFT 预热效果更好，并引用了原文 Table 7 的数据，这一提炼有助于避免读者盲目套用传统流程。
挑刺:
- SWE-Bench 提升数字不准确。 博文称“在更通用的代码基准（如 SWE-Bench Verified）上仅提升了约 5 个点”，但原文 Table 4 显示 Qwen 3.5 9B 的 SWE-Bench Verified 得分为 44.0±2.0，TMAX-9B 为 53.5±0.6，实际提升 9.5 个点。原文表述为“improves… by over 5 points”，博文将其简化为“约 5 个点”且加“仅”字，低估了泛化幅度。
- 遗漏了不同模型尺寸的实验结果。 博文全文仅聚焦 9B 模型，未提及原文 Table 3 和 Table 16 中 TMAX-2B、TMAX-4B、TMAX-27B 同样显著优于各自基线的数据（例如 TMAX-27B 在 TB 2.0 上达到 42.7%，超越 Qwen 3.6 27B 的 39.6%）。这可能导致读者误以为该配方仅对 9B 有效。
