# ⭐⭐⭐⭐ 精神病问诊评测：LLM 问诊为何“问得好”却“诊不准”？

**日期**: 2026-06-24

---

论文 : LingxiDiagBench: A Multi-Agent Framework for Benchmarking LLMs in Chinese Psychiatric Consultation and Diagnosis链接 : https://arxiv.org/abs/2602.09379在医疗 AI 领域，大多数评测还停留在“给一段病历让模型下诊断”的静态模式。但这完全脱离了现实：精神科医生是通过多轮问诊、逐步收集信息来确诊的。这篇来自 EverMind AI 和上海市精神卫生中心的论文，构建了一个名为 LingxiDiagBench 的多智能体（Multi-Agent）评测框架，首次在中国语境下模拟了真实的动态精神科问诊过程。
### 为什么现有的评测不够用？
目前的精神健康 AI 基准测试存在三个致命缺陷：
- 对话太假：多数基于模板生成，缺乏真实临床对话的变异性。
- 标签缺失：缺少经过临床医生验证的诊断金标准（Golden Standard）。
- 无法动态交互：不支持多轮问诊，导致无法评估模型的信息收集策略是否有效。
### 核心设计：用 Agent 模拟“医患博弈”
论文的核心 Insight 在于将问诊拆解为三个独立的 Agent 角色，从而解耦了“患者表现”、“医生提问策略”和“诊断推理能力”。
-患者 Agent (LingxiDiag-Patient)：
基于 16,000 条合成电子病历（EMR）驱动。作者发现直接让 LLM 扮演患者会导致回答过长、症状一次性倒出等不自然现象。因此，他们通过控制回复长度分布和特定的 Prompt 工程，模拟了真实患者的“保留性”和“口语化”。评测显示，Qwen3-32B 驱动的患者 Agent 在真实性上甚至超过了部分真实临床数据基线（Overall Score 4.67 vs 4.30）。
-医生 Agent (Doctor Agent)：
这是评测的重点。作者对比了四种问诊策略：
Free-form：让 LLM 自由发挥，模拟资深专家直觉。
- Symptom-Tree：基于症状决策树，但难以扩展。
- APA-Guided：遵循美国精神医学协会（APA）的五阶段指南（筛查、评估、深入、风险、结案）。
- APA-Guided + MRD-RAG：在指南基础上，检索 Top-3 潜在诊断指南辅助提问。
-诊断 Agent (Diagnosis Agent)：
基于完整的对话历史，依据 ICD-10 标准进行证据推理并给出诊断。
### 关键发现：问诊质量 ≠\neq= 诊断准确率实验结果揭示了几个反直觉的现象，这对工程落地极具参考价值。
1. 难度阶梯下的性能断崖在静态评测中，LLM 的表现尚可，但在动态多轮问诊中，性能随疾病类别数量增加而急剧下降：
- 二分类（抑郁 vs 焦虑）：准确率高达 92.3% (DeepSeek-V3.2 + APA-Guided + RAG)。
- 四分类（含共病）：准确率跌至 43.0%。
- 十二类多标签诊断：准确率仅 28.5%，Top-1 准确率最高也才 37.5%。
这说明 LLM 在处理复杂鉴别诊断时，极易受到噪声干扰或信息收集不全的影响。
⚠️ 反直觉发现 ：LLM-as-a-Judge 评估的“问诊质量”与最终的“诊断准确率”仅呈中度相关（ r=0.43r = 0.43 0.43 ）。
这意味着，一个模型可以问出非常专业、结构清晰的问题（高问诊分），但最终给出的诊断却是错的。 提问技巧并不自动转化为推理能力 ，两者需要分开优化。
2. RAG 策略的双刃剑效应在合成数据上，加入检索增强（MRD-RAG）使整体分类性能提升了 5% 。但在真实临床数据测试中，简单的 Symptom-Tree 策略反而优于复杂的 APA-Guided + RAG。这表明，对于典型病例，过度复杂的检索可能会引入噪声或偏离核心症状路径。
### 工程启示- 不要迷信静态准确率：如果你的应用场景是辅助问诊，仅看静态病历诊断的准确率会严重高估模型能力。必须引入动态交互评测。
- 解耦优化策略：在构建医疗 Agent 时，应将“提问策略”（如何问）和“诊断推理”（如何判）视为两个独立模块。目前的 SOTA 模型在“问”上已经不错，但在“诊”的鲁棒性上仍有巨大缺口。
- 患者模拟的重要性：真实的患者往往不会配合地抛出所有症状。构建高保真的 Patient Agent 是训练和评估 Doctor Agent 的前提，否则会出现“过拟合完美患者”的问题。
### 局限与展望尽管框架先进，但目前的 Patient Agent 仍难以覆盖个体患者的全部多样性。此外，动态问诊中信息收集策略的低效会显著拖累下游诊断，这提示我们未来可能需要引入强化学习（RL）来专门优化医生的提问路径，而不仅仅是依赖预训练模型的直觉。
