论文 : CEO-Bench: Can Agents Play the Long Game?
链接 : https://arxiv.org/abs/2606.18543现在的 AI Agent 在修 Bug、写脚本这些“短平快”任务上已经挺溜了。但现实世界不是做题，而是连续决策：预算怎么投？产品怎么改？市场变了怎么办？这篇论文直接给 Agent 扔了一个硬核挑战：模拟经营一家初创公司 500 天。
### 为什么现有评测不够用？
现有的 Agent 评测（如 SWE-bench、WebArena）大多关注“单次任务执行”。目标明确，反馈即时，环境静态。这就像让厨师切菜，切好了就完事。但做 CEO 不同：
- 长周期：决策后果滞后，今天投广告，下个月才见转化。
- 噪声大：没有上帝视角，只能从混乱的数据库、社交媒体和财务报表中推断真相。
- 动态博弈：竞争对手在动，宏观经济在变，客户偏好也在漂移。
CEO-Bench 的核心洞察是： 真正的智能体现在“不确定性下的长期战略协调” ，而不仅仅是工具调用的准确率。
### 方法拆解：一个残酷的模拟沙盒作者构建了一个名为 NovaMind 的虚构 SaaS 公司模拟器。Agent 通过 Python API 操作，拥有 34 种工具和 19 张业务数据库表。
设计亮点在于“反作弊”机制：
- 微观仿真：不模拟 aggregate demand（总需求），而是模拟 26 个客户群、每个群里独立的个体。每个客户有独特的价格敏感度、质量阈值和满意度曲线。
- 机械式规则：不用 LLM 当裁判，所有结果由明确的数学公式驱动。比如客户是否续费，取决于“感知质量”与“价格”的微观经济学权衡。
- 部分可观测：Agent 看不到“客户满意度”这个数值，只能看社交媒体抱怨、取消率和工单量来反推。
这种设计迫使 Agent 必须像真人一样： 写 SQL 查数据 -> 分析趋势 -> 制定策略 -> 执行工具 -> 等待反馈 。它测试的是 Agent 的“信息获取-推理-行动”闭环能力。
### 关键结果：只有两个模型没破产实验结果非常残酷。在 500 天的模拟中，大多数 SOTA 模型都宣告破产（Cash < 0）。

行为模式差异显著：
- Claude Opus 4.8 & GPT-5.5：表现出活跃的探索性。它们会写代码模拟用户队列、预测现金流，并根据市场变化动态调整广告和研发预算。
- Claude Opus 4.7：采取“保守保命”策略。遇到挫折就砍预算、停广告，虽然活到了最后，但利润微薄（仅 $39 万），远低于规则基线。
### 工程启示- Agent 需要“记忆”与“规划”：在长周期任务中，简单的 ReAct 循环不够用。Agent 需要维护一个可编辑的记忆文件（Memory File），记录历史决策和假设，以便跨周次进行战略调整。
- 代码即策略：最强的 Agent 不是直接调 API，而是写 Python 脚本批量处理数据、模拟未来情景。这提示我们，赋予 Agent 完整的编程环境比提供固定工具更重要。
- 噪声处理能力是瓶颈：大多数模型死于无法从噪声中识别信号。例如，Grok 和 Gemini 在早期就因误判市场而耗尽现金。
### 局限与展望CEO-Bench 目前主要测试单一公司的运营，未涉及复杂的供应链或多主体博弈。但作为一个基准，它清晰地划出了当前 Agent 能力的边界： 执行没问题，战略很欠缺 。对于想构建长期自主 Agent 的工程师来说，这是一个必须跨越的鸿沟。
