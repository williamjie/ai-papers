**日期**: 2026-06-24

论文 : AGORA: An Archive-Grounded Benchmark for Agentic Workplace Document Reasoning链接 : https://arxiv.org/abs/2606.24526企业级 AI Agent 的终极战场不在聊天框，而在杂乱无章的内部文档库。
复旦团队发布的 AGORA 基准测试揭示了一个残酷现实：即便最强模型，在真实职场文档推理任务中的准确率也不足 60%。这不仅是 RAG 检索的问题，更是 Agent 规划与证据整合能力的全面考验。
### 现有评测的“温室效应”
目前的 LLM 评测大多存在“温室效应”。
HotpotQA 等多跳问答基准依赖维基百科等结构化语料；FinanceBench 仅覆盖单一金融领域；GAIA 等 Agent 基准则允许开放网络搜索。这些场景要么过于整洁，要么缺乏对内部私有数据的严格限制。
真实职场文档具有三个致命特征：
- 非结构化与异构：PDF、Excel、Markdown 混排，术语不统一。
- 证据稀疏且分散：关键线索隐藏在海量无关文件中，无法通过简单关键词匹配定位。
- 上下文溢出：单个域的数据量（如 AGORA 中的金融领域达 1.17 亿 Token）远超模型上下文窗口，强制 Agent 进行主动探索而非全量扫描。
AGORA 正是为了填补这一空白，构建了一个包含 8 个专业领域、9,664 份真实文档、3.72 亿 Token 的封闭基准。
### 核心设计：为什么 AGORA 更难？
AGORA 的核心 Insight 在于将“文档推理”重新定义为**“基于档案的主动探索与证据调和”**。
其构建流程极具工程参考价值：
- 数据清洗标准化：利用 dots.ocr 将 PDF 转为 Markdown，Excel/CSV 解析为紧凑表格配置，统一为纯文本供 Agent 读取。
- 防泄漏混淆（Obfuscation）：这是关键创新。传统基准常因实体名称直接出现在问题中导致“检索捷径”。AGORA 通过重写实体描述（如将具体公司名替换为业务角色描述），强制 Agent 必须理解上下文逻辑而非依赖词汇匹配。
- 难度过滤：剔除仅靠模型参数记忆即可回答的问题，确保所有答案必须从挂载的本地档案中推导得出。
每个任务要求 Agent 仅通过 bash 工具调用（如 ls , grep , cat ）在隔离沙箱中探索文件结构，最终输出唯一确定的数值答案。这种设计剥离了复杂 Agent 框架的干扰，纯粹测试模型的推理与工具使用能力。
### 实验结果：最强模型也未及格在 mini-swe-agent 极简框架下，8 款前沿模型的表现令人沮丧。

⚠️ 反直觉发现 ：整体排名与领域表现严重错位。
Gemini-3.1-Pro 虽总分第一，但在金融领域仅得 41%，远低于 GLM-5.1 和 GPT-5.5。GPT-5.5 在法律和技术领域领先，但在商业领域垫底。这说明**“模型能力”并非通用属性，而是与特定领域的文档结构强相关**。
小模型表现更惨烈：Qwen3.5-9B 在五个领域中准确率低于 3%，Gemini-3.1-Flash-Lite 在六个领域低于 7%。这意味着低成本模型在当前架构下几乎无法胜任此类任务。
### 失败模式分析：不是算不对，是找不到论文对错误类型进行了精细标注，揭示了 Agent 的主要瓶颈：
- 检查不完整（Incomplete Inspection）：Agent 跳过了包含关键证据的文件。
- 证据误识（Evidence Misidentification）：找到了文件，但提取了错误数值或误解了单位/时间约定。
- 资源耗尽（Resource Exhaustion）：GPT-5.5 的失败中 24.59% 源于此，而 Gemini-Flash-Lite 高达 69.61%。这表明小模型极易陷入无效循环，耗尽 Token 或时间预算。
值得注意的是， 幻觉（Hallucination）在前沿模型中占比低于 12% 。这说明主要矛盾不在于“编造答案”，而在于**“证据定位与整合”**的失败。Agent 缺乏在海量噪音中精准导航的能力。
### 工程启示对于正在构建企业级文档 Agent 的工程师，AGORA 提供了三点明确指导：
- 不要迷信长上下文：当数据量超过上下文窗口时，单纯的 stuffing 策略失效。必须设计分层检索与主动探索机制，让模型学会“先看目录，再读细节”。
- 领域适配至关重要：没有通用的最强模型。在金融场景可能需要强化数值对齐能力，在法律场景则需加强条款逻辑推理。微调或 Prompt 工程应针对特定领域的文档结构进行优化。
- 简化 Agent 接口以聚焦核心能力：AGORA 使用极简 bash 接口证明了复杂框架并非必需。在初期调试中，剥离冗余工具链有助于定位模型本身的推理缺陷。
### 局限与展望AGORA 目前仅覆盖 8 个领域，且查询数量为 362 条，虽保证了质量但规模有限。此外，随着模型预训练数据的增长，部分“私有”文档可能被泄露进参数空间，未来需持续更新基准以维持挑战度。
总之，AGORA 告诉我们：让 LLM 成为可靠的职场助手，路还很长。当前的瓶颈不在生成流畅文本，而在像人类专家一样，在杂乱档案中抽丝剥茧、去伪存真。
