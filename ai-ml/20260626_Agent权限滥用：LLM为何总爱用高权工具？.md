# Agent 权限滥用：LLM 为何总爱用高权工具？

**日期**: 2026-06-26

---

论文 : When Lower Privileges Suffice: Investigating Over-Privileged Tool Selection in LLM Agents链接 : https://arxiv.org/abs/2606.20023在 Agent 落地过程中，我们常忽略一个隐患：Agent 并非“做不到”，而是“做得太宽”。这篇论文揭示了 LLM 在工具选择上的“过度特权”倾向——明明低权限工具能完成任务，模型却倾向于调用高权限接口。这种偏好不仅增加攻击面，更可能在生产环境中引发不可控的副作用。
### 痛点：安全对齐没拦住“越权”操作现有的 Agent 安全研究多聚焦于恶意指令（如注入、越狱），或系统级的外部访问控制。但论文指出， 常规的安全对齐无法迁移到“最小权限原则” 。
实验显示，使用 AgentAlign 进行安全微调后，Ministral-8B 的危害分数从 67.4% 降至 10.5%，但其过度特权工具使用率（OPUR）仅微降 6.3%；Qwen2.5-7B 甚至出现了 OPUR 上升的情况。这说明：教模型“拒绝有害请求”，并不等于教它“在合法选项中选最保守的那个”。
### 核心洞察：失败是越权的催化剂论文构建了 ToolPrivBench ，通过模拟环境剥离了“低权限工具能力不足”的干扰变量。每个场景提供 3 个低权限和 3 个高权限工具，且所有工具均能独立完成任务。
关键发现在于 临时故障对模型决策的放大效应 ：
⚠️ 反直觉结论 ：当低权限工具遭遇非特权相关的临时错误（如连接超时）时，Agent 会迅速放弃保守策略，转向高权限工具。
以 GPT-5.2 为例，零样本直接越权（PED=0）仅发生 5 次；但在经历一次低权限失败后（PED=1），越权次数升至 13 次；两次失败后（PED=2）则激增至 35 次。这种“挫败感”导致模型产生能力不确定性，从而选择更“强力”的高权方案作为兜底。
### 数据拆解：谁最爱滥用权限？
论文评估了 11 个主流模型，OPUR 差异显著：

### 工程解法：特权感知后训练既然 Prompt 工程和通用安全对齐效果有限，论文提出了一种**特权感知的后训练（Privilege-Aware Post-Training）**方案：
- SFT 阶段：构建理想轨迹，演示如何区分“临时故障”与“能力不足”，并在 <think>...</think> 中显式推理权限边界。
- RL 阶段：使用 GRPO 优化，奖励函数明确偏好“低权限成功”，惩罚“未充分探索即升级”。
效果显著：Qwen3-8B 的 OPUR 从 50.4% 降至 27.02%，Qwen3-4B-Thinking 更是降至 18.93%。且该干预对通用能力（MMLU、GSM8K）影响极小，保留率均在 99% 以上。
### 启示与局限对工程师而言，这篇论文的价值在于提醒我们： 不要假设 Agent 会默认遵循最小权限原则 。在部署 Agent 时：
- 监控升级路径：重点监控低权限工具失败后的高权切换行为，这往往是风险高发点。
- 针对性微调：若使用开源模型，需引入特权感知数据进行 SFT/RL 训练，而非仅依赖通用安全对齐。
- 容忍临时故障：在系统设计上，应允许 Agent 在低权限层面进行多次重试，避免因单次超时触发不必要的权限升级。
局限在于当前评估基于模拟环境，未涉及真实生产中的复杂工具链和长程依赖。未来需进一步研究在多工具协作场景下的权限隔离机制。
