# ⭐⭐⭐½ 拒绝偷懒：MLLM 蒸馏中的感知推理解耦

**日期**: 2026-06-18

论文 : Seeing Before Reasoning: Decoupling Perception and Reasoning for Shortcut-Resilient Multimodal On-Policy Self-Distillation链接 : https://arxiv.org/abs/2606.19120在 MLLM 后训练领域，On-policy Self-Distillation (OPSD) 凭借高密度的 Token 级监督信号，已成为提升推理能力的高效手段。然而，直接将其从纯文本迁移到多模态场景时，模型极易陷入“走捷径”的陷阱：为了迎合已知答案而忽略图像细节。
这篇来自中科院沈阳自动化研究所的工作 ViGOS，通过强制模型“先看后想”，巧妙地在蒸馏过程中解耦了感知与推理，有效抑制了这种幻觉式的偷懒行为。
### 痛点：OPSD 在多模态下的“先入为主”
标准 OPSD 的核心在于利用一个冻结的、知晓正确答案的教师模型（Teacher），对学生模型（Student）生成的轨迹进行逐 Token 的修正。在纯文本逻辑推理中，这非常有效。但在 MLLM 中，问题变得微妙。
由于语言先验（Language Prior）的强大，模型往往更倾向于依赖文字线索而非图像内容。当教师模型能看到参考答案 a∗a^* 时，它会在学生生成早期 Token 时就提供强烈的“答案导向”信号。这导致学生学会了一种捷径：直接构建符合答案的逻辑路径，而无需真正去“看”图。
作者定义了一个诊断指标 PALR（Privileged Answer Leakage Rate）来量化这种泄露。在 Qwen2.5-VL-3B 上，Vanilla OPSD 的推理阶段 PALR 高达 17.26% ，意味着近两成的监督信号纯粹由答案驱动，而非图像证据。
### 方法拆解：强制“看见”再“思考”
ViGOS (Visual Grounding On-Policy Self-Distillation) 的核心直觉很简单： 不要让答案污染早期的视觉感知。
它通过结构化输出和分段监督，实现了感知与推理的物理隔离：
- 结构化生成：学生必须按 <description> d </description> -> <think> r</think> -> a 的顺序生成。
- 分段教师监督：
描述阶段 (dd)：使用一个仅见图像的感知教师（Perception Teacher）。它看不到问题文本和答案，只能基于图像指导学生写出准确的视觉描述。这强制模型建立扎实的视觉 grounding。
- 推理阶段 (r,ar, a)：在描述生成后，切换为拥有参考答案的特权推理教师（Reasoning Teacher）。此时，答案信号才介入，指导后续的逻辑推导。
这种设计的妙处在于，它保留了 OPSD 的高密度监督优势，但严格控制了信息流入的时机。答案引导依然存在，但它只能作用于已经确立视觉事实之后的推理过程，无法再诱导模型编造视觉细节。
### 关键结果：既快又稳实验表明，ViGOS 在保持甚至超越标准基准性能的同时，显著提升了抗干扰能力。
1. 基准测试表现优异在 Qwen2.5-VL-7B 上，ViGOS 的平均 Pass@5 达到 75.60% ，优于 Baseline (68.13%) 和 Vanilla OPSD (75.47%)。特别是在需要强视觉理解的 RealWorldQA 和 CV-Bench 上，提升明显。
2. 捷径泄露大幅降低PALR 指标显示，ViGOS 将推理阶段的 PALR 从 26.01% (7B) 降至 7.56% 。这意味着监督信号变得更加“干净”，真正基于视觉证据。
3. 对抗语言先验的能力更强在 ViLP 基准（测试模型在图文冲突时是否信图）上，ViGOS-7B 的 Score 达到 62.17 ，显著高于 Baseline (39.50) 和 OPSD (58.33)。更关键的是，它保持了极高的 Prior 准确率（~91-97%），说明它没有为了看图而丢弃有用的常识，而是学会了在冲突时正确取舍。
### 工程启示对于从事 MLLM 微调的工程师，ViGOS 提供了几个极具价值的实践指导：
- 结构化输出是控制流的关键：通过强制模型先生成中间态（如视觉描述），我们可以对训练过程进行更精细的干预。这比端到端黑盒蒸馏更具可控性。
- 教师模型的上下文隔离：在蒸馏中，不必让 Teacher 拥有所有信息。根据训练阶段动态调整 Teacher 的可见上下文（如早期屏蔽答案），可以有效防止模型学到错误的因果关联。
- 低成本改进：ViGOS 仅需修改训练时的 Prompt 和 Loss 计算逻辑，无需引入复杂的 RL 机制或额外的外部教师模型，工程落地成本极低。
### 局限与展望当然，方法也有边界。生成的视觉描述 dd 本身可能存在噪声或不完整，且额外的 Teacher Forward Pass 增加了计算开销。未来，如何优化描述的质量，以及如何将这种解耦思想扩展到视频或多轮对话场景，值得进一步探索。
总之，ViGOS 提醒我们：在追求推理性能时，不要忽视“感知”这一基础环节。让模型先看清世界，再思考答案，才是稳健的多模态智能之路。
