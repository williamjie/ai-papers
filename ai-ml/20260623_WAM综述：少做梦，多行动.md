# ⭐⭐⭐⭐ WAM综述：少做梦，多行动

**日期**: 2026-06-23

---

论文 : World Action Models: A Survey链接 : https://arxiv.org/abs/2606.20781embodied AI（具身智能）圈子里，“世界模型”这个词被滥用得太厉害了。视频生成模型、VLA策略、模拟器，全都被塞进这个筐里。这篇来自新加坡国立大学的综述《World Action Models: A Survey》干了一件极其实用的事：它重新定义了**世界行动模型（WAM）**的边界，并给出了一个清晰的工程分类法。
对于工程师来说，核心价值不在于看热闹，而在于搞清楚： 在控制回路中，到底该“做梦”到什么程度？
### 痛点：VLA 缺乏因果，视频生成太慢现有的 VLA（Vision-Language-Action）模型，如 RT-2 或 OpenVLA，本质上是 p(a∣o,l)p(a|o, l) 。它们从当前观测直接映射到动作。
缺陷很明显 ：它们不预测环境变化。没有物理引擎，没有接触反馈，也没有视角变化的显式建模。这导致策略缺乏对“后果”的推理能力。
于是大家转向世界模型（World Models），试图预测 p(o′∣o,a,l)p(o'|o, a, l) ∣ o , a , l ) 。但直接套用大规模视频生成模型（如 Sora、Wan）做控制， 延迟太高，算力太贵 。
这就引出了 WAM 的核心定义： 预测的未来必须进入动作路径 。无论是用来产生动作、评分动作，还是在联合训练中指导动作，未来观测 o′o' 必须是动作决策的一部分。
### 核心洞察：三种设计哲学论文没有停留在罗列工作，而是根据**“在推理路径的哪里解码动作”**，将 WAM 拆解为三类。这个分类直接对应了工程上的算力与精度权衡：
-Render-and-Decode（渲染并解码）
逻辑：视频生成 backbone 跑到底，输出像素，再从像素里提取动作。
- 代表：UniPi, GR-1/2, DreamZero。
- 优点：保留了完整的视觉先验，可解释性强（你能看到机器人“想”的画面）。
- 缺点：慢。每一步都要付完全的去噪或自回归代价。
-Latent-Only（仅潜空间）
逻辑：沿用视频生成 backbone，但在像素解码前截断。从中间层的 latent、flow field 或 mask 中提取动作。
- 代表：VPP, Fast-WAM, mimic-video。
- 优点：快。省去了最耗时的 VAE 解码步骤，保留了动态结构先验。
- 缺点：失去了像素级的监督信号，调试难度增加。
-Video-Generation-Free（无视频生成）
逻辑：彻底抛弃视频生成 backbone。在 LLM/VLM 的 token 空间、JEPA 特征空间或音频空间进行预测。
- 代表：DUST, Audio-WM, HiF-VLA。
- 优点：极快，计算开销最低。
- 缺点：缺乏显式的视觉动态建模，泛化性依赖语言/特征空间的抽象能力。
⚠️ 关键结论 ：领域正在从“渲染更多未来”转向**“生成更少未来，但保留控制所需的关键信息”**。这不是技术倒退，而是工程必然。
### 方法拆解：四维解剖法为了更细致地对比，论文提出了一个 4-tuple 标记法，任何 WAM 都可以被描述为：
(Predictive Substrate, Architectural Backbone, Action Coupling, Deployment Regime)
- 预测基底 (Substrate)：未来长什么样？像素、特征、几何原语，还是亲和图（Affordance map）？
- 架构骨干 (Backbone)：怎么算出来的？扩散模型、自回归、JEPA 还是混合？
- 动作耦合 (Coupling)：动作怎么进来？是条件化 rollout、联合生成，还是预测后加个 action head？
- 部署模式 (Deployment)：开环、分块闭环（Chunked closed-loop），还是单步闭环？
为什么这么设计？
因为不同的基底决定了信息��丰富度，而耦合方式决定了延迟。例如，GR-1 使用自回归 backbone 联合预测图像 token 和动作 token，虽然耦合紧密，但串行生成限制了速度；而 Fast-WAM 通过特征 tap 在训练时共享未来分支，推理时跳过渲染，实现了速度与性能的平衡。

### 工程启示- 别盲目追求像素级未来：如果你的机器人需要毫秒级响应，直接上视频生成模型做 WAM 是自杀。考虑 Latent-Only 或 Video-Generation-Free 方案。
- 耦合方式决定延迟上限：如果是闭环控制，尽量采用“分块预测（Chunked closed-loop）”或“特征 tap”，避免每一步都重新推理整个未来序列。
- 数据是关键瓶颈：WAM 需要 (o,a,o′)(o, a, o')) 三元组数据。现有的互联网视频缺乏动作标签，遥操作数据规模太小。如何利用仿真数据或自监督信号（如 JEPA）来预训练预测基底，是落地的关键。
### 局限与展望论文最后指出了几个开放挑战：
- 记忆问题：长 horizon 预测中，模型如何保持状态一致性？
- 物理合理性：生成的未来是否符合物理定律？目前的评估主要靠视觉相似度，缺乏物理约束。
- 抽象动作的 grounding：当动作空间从关节角度变为高层语义时，WAM 如何保证预测的有效性？
总的来说，这篇综述为混乱的 WAM 领域提供了急需的“地图”。它告诉我们： 未来的方向不是生成更逼真的视频，而是生成更高效的控制信号。 Dream Less, Act More.
