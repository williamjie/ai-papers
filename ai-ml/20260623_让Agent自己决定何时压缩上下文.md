**日期**: 2026-06-23

论文 : Self-Compacting Language Model Agents链接 : https://arxiv.org/abs/2606.23525现在的长程 Agent 就像个记性越来越差的老头。随着推理链和工具调用不断累积，上下文窗口里塞满了过时的中间结果、废弃的代码片段甚至错误的假设。这种现象被称为“上下文腐烂”（Context Rot），它会像锚点一样拖垮后续的生成质量。
现有的工程方案通常很粗暴：要么等到 Token 用完了再被动压缩，要么每隔固定步数就强制总结。问题在于，这种基于时间或长度的触发器完全无视了任务的逻辑结构。在模型推导到一半时强行打断并总结，往往会把关键的中间变量给“洗”掉，导致模型不得不重新搜索甚至产生幻觉。
这篇论文提出了 Self-Compact ，核心思路很简单：别用死板的定时器，让模型自己决定什么时候该压缩上下文。但这不仅仅是给模型加个 <summarize> 工具那么简单，因为实验发现，如果只给工具不给指导，模型要么乱压缩，要么根本不压缩。
### 核心 Insight：Rubric-Gated 自适应压缩Self-Compact 的设计由两个推理时（Inference-time）的组件组成，缺一不可：
- 内联压缩工具：模型可以主动调用该工具来总结当前上下文。
- 轻量级评判标准（Rubric）：这是灵魂所在。它规定了何时触发压缩（子任务已解决、轨迹正在收敛），以及何时抑制压缩（推导中途、陷入死循环）。
关键直觉 ：模型本身缺乏“元认知”能力，它不知道自己的上下文什么时候开始腐烂。通过提供一个轻量级的 Rubric，我们实际上是在推理时向模型注入了这种能力，而无需任何微调（Fine-Tuning）。
具体实现上，系统会在固定的探针间隔（Probe Interval）询问模型当前状态。如果 Rubric 判定为 COMPRESS ，则执行总结并替换上下文；否则继续生成。由于利用了 KV Cache 复用，这种探针带来的额外计算开销极小。

### 工程启示对于正在构建长程 Agent 的工程师来说，这篇论文有几个直接可用的建议：
- 不要迷信固定阈值：如果你现在的系统是按 Token 数量或步数强制总结，请重新评估。在推理中途切断上下文是灾难性的。
- Rubric 比工具更重要：单纯暴露压缩接口给模型是不够的。你需要精心设计 Prompt，明确告诉模型“什么情况下可以安全地丢弃历史”。这需要结合具体任务场景（如数学推导 vs 网页搜索）定制 Rubric。
- 零成本升级：该方法无需微调（SFT/RL），只需修改推理循环和 Prompt。对于使用开源模型（如 Qwen、GLM）的团队，这是一套即插即用的优化方案。
### 局限与展望论文也诚实地指出了局限：目前仅在开源权重模型上验证。前沿闭源模型（如 Claude Opus、GPT-5.5）可能本身具备更强的元认知能力，不需要额外的 Rubric 就能判断何时压缩。此外，该方法尚未结合强化学习（RL），未来可以通过 RL 将这种“何时压缩”的行为蒸馏进模型权重中，进一步降低推理时的探针开销。
总之，Self-Compact 证明了通过简单的推理时脚手架，我们可以显著提升 Agent 的长期记忆管理能力。对于受限于上下文窗口和成本的实际应用，这是一个值得立即尝试的方向。
