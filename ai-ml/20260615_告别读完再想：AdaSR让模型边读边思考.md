# ⭐⭐⭐½ 告别读完再想：AdaSR让模型边读边思考

**日期**: 2026-06-15

论文 : AdaSR: Adaptive Streaming Reasoning with Hierarchical Relative Policy Optimization链接 : https://arxiv.org/abs/2606.14694现在的 LLM 推理大多遵循“读完再想”模式：必须等完整输入结束，才开始生成 Chain-of-Thought（CoT）。这在处理长文本、实时语音或视频流时简直是灾难——用户不得不忍受漫长的首字延迟。AdaSR 提出了一种自适应流式推理框架，让模型在接收输入的同时进行局部思考，并在最后进行深度整合。
### 痛点：现有流式推理只是“假装”思考现有的流式 CoT 方法（如 StreamingThinker）主要依赖监督微调（Supervised Fine-Tuning, SFT）。这种方法的致命伤在于 模仿学习 ：模型只是在机械复现预定义的流式轨迹，缺乏真正的自适应能力。它不知道何时该深入思考，何时该跳过无关信息，更无法动态分配计算资源。
更深层的问题出在强化学习（RL）的优化目标上。标准的 GRPO 算法将整个推理过程视为一个扁平序列，用同一个序列级优势值（Sequence-level Advantage）更新所有 Token。这导致了一个 跨阶段信用悖论 ：深度推理阶段的正确结果，可能会错误地放大流式阶段中冗余的思考；反之，深度阶段的低效也会不公平地惩罚流式阶段的有效探索。
### 核心拆解：HRPO 与分层奖励AdaSR 的核心创新在于 Hierarchical Relative Policy Optimization (HRPO) ，即分层相对策略优化。其设计直觉非常清晰： 不同阶段的 Token 应该承担不同的责任 。
-分层优势分配：
HRPO 将推理轨迹拆分为“流式阶段”（局部理解）和“深度阶段”（全局整合）。它不再使用单一优势值，而是为每个阶段计算独立的优势信号。流式 Token 关注局部决策的正确性，深度 Token 关注最终答案的准确性，而全局优势则确保整体任务完成。
-自适应思考奖励：
为了防止模型废话连篇，AdaSR 引入了基于 Token 长度的惩罚机制。但它很聪明地使用了对数形式 -log(1 + Length)，这意味着随着长度增加，边际惩罚递减。同时，它区分了流式延迟和深度延迟：流式思考可以与输入接收重叠，因此其延迟成本被打折（系数 α<1\alpha < 11），而深度推理必须等待所有输入结束，因此承担全额延迟惩罚。
-工程落地优化：
论文基于 vLLM 实现了流式 Rollout，通过独立的 Position ID 和 Attention Mask，确保模型在生成局部思考时无法“偷看”未来的输入。这保证了训练与推理的一致性（Training-Serving Consistency）。
### 实验数据：效率与准确率的双赢AdaSR 在 Qwen3-1.7B 和 Qwen3-4B 上进行了广泛测试，结果令人印象深刻。以下以 GSM-symbolic P2（高难度数学推理）为例，对比 SFT 基线与 AdaSR-HRPO 的表现：

⚠️ 反直觉发现 ：在 Qwen3-4B 上，HRPO 不仅提升了准确率，还 减少了总生成长度（-8.4%） 。这说明更强的基座模型能更好地利用自适应机制，剔除冗余思考，实现帕累托改进。
在延迟方面，得益于流式推理与输入接收的重叠，AdaSR 在 vLLM 后端实现了 8.82倍 的延迟降低（相比 Read-then-think）。这意味着对于长上下文任务，用户几乎可以即时看到模型的初步反应。
### 工程启示：如何落地？
- 粒度选择至关重要：论文对比了 Token 级、句子级和阶段级的优势分配。结果显示，阶段级（Stage-level） 效果最好。过细的粒度（如句子级）会割裂语义连贯性，导致准确率下降。这提示我们在设计 RLHF 奖励时，不必追求极致的细粒度，而应匹配任务的逻辑结构。
- 小模型更需要自适应计算：Qwen3-1.7B 在 HRPO 下虽然总长度略增，但准确率大幅提升（+22.7%）。对于算力受限的边缘设备，这种“按需思考”的机制比固定长度的 CoT 更具性价比。
- 格式奖励是稳定器：实验表明，仅靠准确率奖励会导致训练不稳定。引入严格的格式奖励（如 <EOT> 标记）能显著加速收敛，并提高最终性能。
### 局限与展望目前 AdaSR 主要验证于文本流和可验证答案的任务（如数学、逻辑）。对于开放域对话、音频或视频流，如何定义“局部思考”的边界以及如何设计多模态奖励信号，仍是未解之谜。此外，论文仅使用了 Qwen3 系列模型，其在其他架构上的泛化能力有待进一步验证。
总体而言，AdaSR 为实时推理场景提供了一条清晰的技术路径：通过分层优化解决信用分配难题，让模型真正学会“边读边想”。
