论文 : Lite Any Stereo V2: Faster and Stronger Efficient Zero-Shot Stereo Matching链接 : https://arxiv.org/abs/2606.24457在自动驾驶和机器人领域，立体视觉（Stereo Vision）是获取深度信息的核心手段。然而，现有的高精度模型往往体积庞大，难以部署在资源受限的边缘设备上；而轻量级模型虽然速度快，但在面对未见过的真实场景时，泛化能力通常很差。
这篇论文提出了 Lite Any Stereo V2 (LAS2) ，它打破了“效率与零样本泛化能力不可兼得”的刻板印象。作者不仅重新设计了架构以优化实际推理延迟，还通过三阶段训练策略，让轻量模型在保持极速的同时，拥有了媲美大型基础模型的跨域适应能力。
### 痛点：MACs 不等于速度，合成数据不等于真实世界传统的立体匹配研究往往陷入两个误区。
首先是 评估指标失真 。许多高效模型声称计算量（MACs）很低，但在现代 GPU 或边缘芯片上，由于内存访问瓶颈或非标准算子，实际推理延迟（Latency）并没有显著下降。论文指出，理论上的 MACs 并不能可靠反映真实部署速度。
其次是 数据域鸿沟 。现有的轻量级模型多依赖合成数据训练，缺乏对真实世界复杂纹理、光照变化的鲁棒性。虽然可以使用单目深度基础模型（如 DepthAnything）提供先验，但这会引入巨大的计算开销，违背了“高效”的初衷。
### 方法拆解：为部署而生的架构设计LAS2 的核心洞察在于： 架构设计必须服务于实际硬件延迟，而非理论复杂度。
1. 纯 2D 成本聚合（2D-only Cost Aggregation）
传统的高精度模型常使用 3D 卷积在代价体（Cost Volume）上进行空间 - 视差联合聚合。LAS2 摒弃了昂贵的 3D 聚合模块，转而采用基于 FasterNet 块的纯 2D U-Net 风格聚合网络。
- Why? 3D 卷积在边缘设备上极慢，而 2D 卷积能更好地利用硬件并行性。实验表明，这种设计在保持精度的同时，大幅降低了 H200 和 Orin 上的推理时间。
2. 三阶段训练策略（Three-Stage Training）
为了弥补轻量模型容量不足导致的泛化缺陷，作者设计了一套渐进式训练流程：
- 阶段一（合成监督）：在约 180 万对合成数据上进行标准监督学习，建立基础的立体匹配能力。
- 阶段二（自蒸馏 Self-Distillation）：教师模型接收干净输入，学生模型接收扰动输入。通过特征对齐损失（Feature Alignment Loss），强迫模型学习域不变表示（Domain-invariant Representations），提升鲁棒性。
- 阶段三（真实世界知识蒸馏）：利用 50 万对无标注的真实世界图像，由强大的教师模型（FoundationStereo）生成伪标签进行微调。
3. 伪标签过滤与误差钳位（Pseudo-label Filtering & Error Clamping）
直接使用伪标签训练容易引入噪声。LAS2 引入了两个关键机制：
- 多掩码过滤：结合左右一致性检查、边缘感知掩码和天空分割掩码，剔除几何不一致或无纹理区域的不可靠标签。
- 误差钳位（Error Clamping）：在损失函数中设置上限，防止少数高误差像素主导梯度更新，确保从合成数据到真实数据的平滑过渡。
### 关键结果：又快又准的 SOTALAS2 提供了四个变体（S, M, L, H），其中 LAS2-M 和 LAS2-H 最具代表性。以下是核心对比数据（基于 KITTI 2015 D1 指标及延迟）：

- 精度超越：迭代版 LAS2-H 不仅精度超过了 Fast-FoundationStereo（D1 从 2.90% 降至 2.64%），而且速度分别是后者的 1.8x (H200) 和 2.7x (Orin)。
- 零样本泛化：在 KITTI 2012、ETH3D 和 Middlebury 等四个真实基准测试中，LAS2-M 均优于之前的 SOTA 前馈高效方法。
### 工程启示对于从事端侧 AI 部署的工程师，这篇论文提供了两个重要指导：
- 警惕 MACs 陷阱：在优化模型时，不要只看 FLOPs/MACs。应直接测量目标硬件上的端到端延迟，优先选择内存访问友好、算子标准的架构（如纯 2D 卷积）。
- 善用未标注数据：通过强教师模型的伪标签结合严格的过滤机制（如左右一致性检查），可以利用海量互联网无标注图像显著提升轻量模型的泛化能力，无需昂贵的真实世界标注。
### 局限与展望尽管 LAS2 表现优异，但论文也指出，在极端天气或极度低纹理区域，性能仍有提升空间。此外，虽然 LAS2-H 通过迭代提升了精度，但其计算量仍高于前馈变体，在算力极受限的场景下需权衡选择。未来，如何进一步压缩迭代模块或在更广泛的域外场景（如水下、夜间）保持鲁棒性，仍是值得探索的方向。
