论文 : DeXposure-Claw: An Agentic System for DeFi Risk Supervision链接 : https://arxiv.org/abs/2606.19501DeFi 的链上数据是噪音的重灾区。如果直接让 LLM 去读原始交易，它大概率会“过度解读”弱信号，导致风控系统频繁误报。这篇论文提出了一种反直觉的工程架构： 不让 LLM 看数据，而是让它看预测后的结构化证据 。
### 痛点：LLM 在金融场景下的“幻觉”陷阱现有的 DeFi 风险监测系统面临两个核心问题。第一，通用 LLM Agent 在处理海量链上数据时，容易把陈旧或不完整的证据当成高风险信号，触发不必要的干预。第二，传统的系统性风险评估往往基于“相对暴露变化”，这会导致大量微小协议被高估，而真正具有系统重要性的头部风险却被忽视。
简单来说，直接让 AI 做决策是不安全的。我们需要一种机制，既能利用 LLM 的推理能力，又能通过确定性逻辑约束它的边界。
### 方法拆解：四层管道与预测接地DeXposure-Claw 的核心设计直觉是 “预测接地”（Forecast-Grounding） 。系统将决策过程解耦为四个层级，LLM 仅作为第三层的“起草者”，而非最终决策者。
- Layer 1 预测：使用图时间序列基础模型 DeXposure-FM，预测未来 1、4、8、12 周的信用暴露网络。这不仅是点估计，还包含蒙特卡洛采样带来的不确定性分布。
- Layer 2 监控与压力测试：这是关键的一步。系统将预测结果转化为结构化的证据包。包括五个监控指标（如 PageRank、HHI集中度）和五种压力场景（如稳定币脱锚、桥接集群故障）。只有当预测统计量偏离滚动基线超过 2 个标准差时，才会触发警报。
- Layer 3 LLM 起草：LLM 接收到的不是原始数据，而是包含警报类型、归因信号、场景损失和不确定性估计的 JSON 载荷。它负责生成带有严重等级和建议的行动票（Ticket）。
- Layer 4 安全门控：引入数据健康度（Data-Health）和置信度（Confidence）两道闸门。如果数据质量低于阈值或警报置信度不足，系统强制进入“安全模式”，仅输出低级别监控建议。
这种设计的精髓在于： LLM 负责解释和排序，确定性模型负责过滤和验证 。
### 关键结果：覆盖率提升，但误报依然存在论文在 DeXposure-Bench 基准上进行了详细评估，结果揭示了一些反直觉的发现。

### 工程启示：金融 AI 的正确打开方式对于量化金融工程师而言，这篇论文提供了明确的落地指南：
- LLM 不是预言机：在高 stakes 场景下，不要让 LLM 直接处理原始信号。先通过统计模型或基础模型进行特征提取和预测，再将结构化证据喂给 LLM。
- 引入确定性门控：必须建立基于数据质量和置信度的硬阈值。LLM 的输出应被视为“建议草稿”，最终释放需经过规则引擎的二次校验。
- 评估指标要务实：不要只看准确率，要关注“虚假干预率”（False-Intervention Rate）。在风控系统中，一次错误的紧急干预可能比漏报一个小风险更具破坏性。
### 局限与展望目前该系统仅针对 DeFi 协议间的信用暴露，且运行频率为周级。对于 Terra/Luna 这种 48 小时内崩塌的黑天鹅事件，周级监控显然滞后。未来若能结合更密集的快照摄入和亚小时级的校准，其工程价值将大幅提升。但无论如何，**“预测+证据+门控”**的架构范式，值得所有金融 AI 系统借鉴。
